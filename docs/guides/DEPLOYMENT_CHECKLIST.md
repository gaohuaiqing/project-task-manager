# 部署检查清单

## 部署前检查

### 环境准备
- [ ] Node.js版本 >= 18.0.0（运行 `node -v` 检查）
- [ ] MySQL已安装并启动（运行 `mysql --version` 检查）
- [ ] Redis已安装并启动（运行 `redis-cli ping` 检查）
- [ ] npm版本 >= 9.0.0（运行 `npm -v` 检查）

### 配置文件
- [ ] `.env` 文件已创建并配置
- [ ] `DB_HOST`、`DB_NAME`、`DB_USER` 已配置
- [ ] `REDIS_HOST`、`REDIS_PORT` 已配置
- [ ] 端口 `PORT=3001` 未被占用

## 安装步骤

### 依赖安装
- [ ] 进入 `app/server` 目录
- [ ] 运行 `npm install`（如有网络问题，参考安装指南）
- [ ] 检查 `node_modules` 目录已创建

### 数据库初始化
- [ ] 运行 `npm run db:init` 初始化数据库
- [ ] 检查数据库表已创建（`projects`, `members`, `wbs_tasks`, `sessions`, `data_versions` 等）
- [ ] 验证默认管理员账号已创建（admin/admin123）

## 服务启动

### 开发模式
- [ ] 运行 `npm run dev` 启动开发服务器
- [ ] 检查控制台输出无错误
- [ ] 验证服务运行在 `http://localhost:3001`
- [ ] 验证WebSocket运行在 `ws://localhost:3001`

### 生产模式
- [ ] 运行 `npm run build` 编译TypeScript
- [ ] 检查 `dist` 目录已创建
- [ ] 运行 `npm start` 启动生产服务器
- [ ] 检查服务正常运行

## 功能验证

### 1. 健康检查
- [ ] 访问 `http://localhost:3001/api/system/health/v2`
- [ ] 验证返回200状态码
- [ ] 检查 `redis.healthy` 为 `true`
- [ ] 检查 `websocket.totalClients` 为数字

### 2. 登录功能
- [ ] 访问系统首页
- [ ] 使用 admin/admin123 登录
- [ ] 验证登录成功

### 3. 刷新免登录
- [ ] 登录后按 F5 刷新浏览器
- [ ] 验证无需重新登录
- [ ] 关闭浏览器重新打开，验证7天内免登录

### 4. 实时同步
- [ ] 在两个浏览器中登录同一账号
- [ ] 浏览器A修改项目状态
- [ ] 验证浏览器B在100ms内自动更新

### 5. 版本冲突控制
- [ ] 在两个浏览器中打开同一项目编辑页面
- [ ] 浏览器A修改并保存
- [ ] 浏览器B修改并保存
- [ ] 验证浏览器B收到409冲突提示

### 6. API端点
- [ ] `GET /api/projects/v2` 返回项目列表
- [ ] `GET /api/members/v2` 返回成员列表
- [ ] `GET /api/tasks/v2` 返回任务列表
- [ ] `GET /api/initial-data/v2` 返回初始数据
- [ ] `GET /api/statistics/v2` 返回数据统计

## 故障排查

### 常见问题处理
- [ ] Redis连接失败 → 系统自动切换离线模式
- [ ] 数据库连接失败 → 检查数据库是否启动
- [ ] 端口被占用 → 检查 3001 端口是否被其他程序占用
- [ ] 会话验证失败 → 检查Cookie是否正确设置
