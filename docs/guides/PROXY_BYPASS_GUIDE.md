# 企业代理问题解决方案

## 🚨 问题描述

**症状：**
- 登录时报错 "工号或密码错误"
- 控制台显示 `ERR_CONNECTION_REFUSED` 或 `Failed to fetch`
- 后端服务正常运行（端口 3001 监听中）

**根本原因：**
您的企业配置了 HTTP 代理服务器 `proxy.mindray.corp:8080`，导致浏览器的所有请求（包括访问本地 `localhost:3001` 的请求）都被转发到企业代理，无法访问本地开发服务器。

---

## ✅ 解决方案（按优先级排序）

### 方案1：配置浏览器代理绕过（最简单，推荐）

#### Chrome/Edge 浏览器

1. 打开浏览器设置
   - 地址栏输入：`chrome://settings/system` 或 `edge://settings/system`

2. 点击 "打开您计算机的代理设置"

3. 在 "代理服务器" 设置中，找到 "绕过代理服务器用于这些主机和域"

4. 添加以下地址（用分号分隔）：
   ```
   localhost;127.0.0.1;10.8.180.55
   ```

5. 点击 "保存"

6. **重启浏览器**，刷新页面

#### Firefox 浏览器

1. 打开设置 → 网络设置
2. 点击 "设置"
3. 在 "无代理" 中添加：`localhost, 127.0.0.1, 10.8.180.55`
4. 点击 "确定"
5. 重启浏览器

---

### 方案2：配置系统环境变量（一劳永逸）

#### Windows 设置

1. 右键点击 "此电脑" → "属性"
2. 点击 "高级系统设置"
3. 点击 "环境变量"
4. 在 "用户变量" 中添加：
   - 变量名：`NO_PROXY`
   - 变量值：`localhost,127.0.0.1,10.8.180.55`

5. 或者直接运行项目根目录的批处理文件：
   ```bash
   fix-proxy.bat
   ```

6. **重启电脑** 或 **注销后重新登录**

---

### 方案3：使用命令行绕过代理（临时方案）

每次启动应用前，在命令行执行：

```bash
# Windows CMD
set NO_PROXY=localhost,127.0.0.1,10.8.180.55
npm run dev

# Windows PowerShell
$env:NO_PROXY="localhost,127.0.0.1,10.8.180.55"
npm run dev

# Git Bash / Bash
export NO_PROXY=localhost,127.0.0.1,10.8.180.55
npm run dev
```

## 📝 常见问题

### Q1: 为什么后端日志显示登录成功，但前端还是失败？

A: 这是因为前端的请求被企业代理拦截了，没有真正到达后端。检查浏览器开发者工具的 Network 面板，会看到请求被发送到代理服务器。

### Q2: 配置了代理绕过后，还能访问外网吗？

A: 可以。代理绕过只对 localhost 和内网地址生效，访问外网（如百度、GitHub）仍然通过代理。

### Q3: 为什么其他同事没有这个问题？

A: 其他同事可能：
- 没有配置企业代理
- 已经配置了代理绕过
- 使用不同的代理设置

## 📞 技术支持

如果问题仍然存在，请检查：

1. 后端服务是否正常运行
   ```bash
   netstat -ano | findstr :3001
   ```

2. 环境变量是否正确配置
   ```bash
   echo %NO_PROXY%
   ```

3. 浏览器代理设置是否生效
