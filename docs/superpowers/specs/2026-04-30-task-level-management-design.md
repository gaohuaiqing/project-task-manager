# 任务层级灵活管理设计

> 日期: 2026-04-30
> 状态: 待审核

---

## 背景

工程师在根任务下创建任务后，经常需要调整任务的层级和顺序。当前系统不支持创建后修改任务层级，误建为子任务后无法纠正。需要一个简单直接的方式让工程师灵活调整任务结构。

## 需求

1. 直接编辑任务的等级值（level），系统自动处理 WBS 编号、parent_id 和排序
2. 拖拽排序调整同级任务的上下顺序
3. 移动任务时连带子任务一起
4. 最大层级限制为 5 层

## 交互设计

### 核心操作：点击等级数字直接编辑

在现有任务列表中，每个任务行已经显示了等级数字（wbs_level）。点击该数字，变为可编辑的输入框。用户输入新等级值后回车确认，系统自动完成所有调整。

**提升层级（等级值减小）**：
- 任务排在原父任务之后，作为同级任务
- 所有子任务跟着一起移动，保持相对层级不变

示例：
```
操作前:                          操作后（竞品分析等级 2→1）:
1  [1] 需求分析                  1  [1] 需求分析
1.1  [2] 用户调研                1.1  [2] 用户调研
1.2  [2] 竞品分析 ← 编辑这个     2  [1] 竞品分析
1.2.1  [3] 用户访谈              2.1  [2] 用户访谈
2  [1] 系统设计                  3  [1] 系统设计
```

**降低层级（等级值增大）**：
- 任务成为上方最近同级任务的子任务
- 所有子任务跟着一起移动

示例：
```
操作前:                          操作后（竞品分析等级 1→2）:
1  [1] 需求分析                  1  [1] 需求分析
2  [1] 竞品分析 ← 编辑这个       1.1  [2] 竞品分析
3  [1] 系统设计                  2  [1] 系统设计
```

### 补充操作：拖拽排序

- 每个任务行左侧有拖拽手柄
- 拖拽只改变同级任务的上下顺序，不改变层级
- 改层级用等级数字编辑

### 边界规则

- 根任务（等级 1）不能再提升
- 第 5 层任务不能再降低
- 等级值限制在 1-5 之间
- 无效输入（如非数字、超出范围）提示错误

## 涉及的关键文件

**前端：**
- `app/src/features/tasks/components/WbsTable.tsx` — 任务列表组件，添加等级编辑和拖拽手柄
- `app/src/features/tasks/index.tsx` — 任务页面入口，处理创建逻辑的联动
- `app/src/features/tasks/components/TaskForm.tsx` — 任务创建表单

**后端：**
- `app/server/src/modules/task/service.ts` — 任务业务逻辑，新增等级变更方法
- `app/server/src/modules/task/repository.ts` — 数据访问层，WBS 重算逻辑
- `app/server/src/modules/task/routes.ts` — API 路由，新增等级变更端点
- `app/server/src/core/types/task.types.ts` — 任务类型定义

## 验证方式

1. 在任务列表中点击等级数字，输入新值，确认后 WBS 编号和缩进正确更新
2. 提升一个有子任务的任务，子任务跟着移动
3. 降低一个任务，它正确成为上方同级任务的子任务
4. 拖拽排序只改变顺序不改变层级
5. 边界情况：根任务不能提升、第 5 层不能降低、无效输入有提示
