# 会话管理功能优化设计文档

> **创建日期**: 2026-04-26
> **状态**: 待实现

---

## 1. 问题背景

### 1.1 现状问题

| 问题 | 描述 |
|------|------|
| 通知内容不清晰 | 用户收到"会话已终止"通知，但不知道原因 |
| 用户无法查看会话 | 前端没有展示会话列表的入口 |
| 开发测试累积会话 | 同一设备多次登录累积会话，触发误通知 |

### 1.2 用户困惑

用户以部门经理身份登录后，收到通知：
```
您的以下设备会话已被终止:
设备: 未知设备
IP: ::1
原因: 登录设备数量超过限制
```

实际上用户只在一台设备上操作，这个通知来源于开发测试过程中的会话累积，导致用户困惑。

---

## 2. 设计目标

1. **优化通知内容**：让用户明确知道发生了什么
2. **新增会话管理页面**：让用户能查看和管理自己的登录设备
3. **提供操作入口**：让用户能主动终止可疑设备

---

## 3. 设计方案

### 3.1 通知内容优化

**触发条件**：仅在 `max_sessions_exceeded` 场景发送通知（被动被挤下线）

**原通知内容**：
```
您的以下设备会话已被终止:
设备: 未知设备
IP: ::1
原因: 登录设备数量超过限制
```

**优化后通知内容**：
```
您在另一台设备登录（IP: xxx），本设备的登录已失效
```

### 3.2 个人资料页面改造

**位置**：设置 → 个人资料页面

**新增区域**：安全设置

**布局结构**：
```
┌─────────────────────────────────────────────────────┐
│  个人资料 Card                                      │
│  - 头像、用户名、显示名称、性别、邮箱、角色          │
├─────────────────────────────────────────────────────┤
│  安全设置 Card                                      │
│  ┌─────────────────────────────────────────────────┐│
│  │ 修改密码                                        ││
│  │ - 当前密码、新密码、确认密码                     ││
│  ├─────────────────────────────────────────────────┤│
│  │ 登录设备                                        ││
│  │ - 设备列表（详见 3.3）                          ││
│  │ - 终止其他所有设备按钮                          ││
│  └─────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘
```

### 3.3 设备列表详情

| 字段 | 说明 | 示例 |
|------|------|------|
| 当前设备标记 | 绿色圆点 + "当前设备" | ● 当前设备 |
| 设备名称 | 解析 User-Agent | Chrome / Windows |
| IP地址 | 脱敏显示（隐藏最后一段） | 192.168.1.\*\*\* |
| 登录时间 | 本地化时间格式 | 2026-04-26 10:30 |
| 最后活跃 | 相对时间 + 绝对时间提示 | 1分钟前 |
| 终止按钮 | 仅非当前设备显示 | 红色按钮 |

**设备列表样式**：
```
┌─────────────────────────────────────────────────────┐
│ ● 当前设备  Chrome / Windows                        │
│   IP: 192.168.1.***                                 │
│   登录: 2026-04-26 10:30    最后活跃: 1分钟前       │
├─────────────────────────────────────────────────────┤
│ ○ 其他设备  Edge / Windows                          │
│   IP: 192.168.1.***                                 │
│   登录: 2026-04-25 18:00    最后活跃: 2小时前       │
│                                    [终止]           │
└─────────────────────────────────────────────────────┘
```

### 3.4 终止操作

#### 单个设备终止
- 点击"终止"按钮
- 弹窗确认："确定要终止该设备的登录吗？"
- 确认后终止该会话

#### 终止其他所有设备
- 点击"终止其他所有设备"按钮
- 弹窗确认："确定要终止其他所有设备的登录吗？将终止 X 个设备"
- 确认后终止除当前设备外的所有会话

## 4. 技术实现要点

### 4.1 后端 API

已有 API，无需新增：
- `GET /api/auth/sessions` - 获取会话列表
- `DELETE /api/auth/sessions/:sessionId` - 终止指定会话
- `POST /api/auth/sessions/terminate-others` - 终止其他所有会话

### 4.2 前端改造

1. **Profile.tsx** 改造
   - 新增"安全设置"区域
   - 移动"修改密码"到安全设置区域
   - 新增"登录设备"组件

2. **新增组件**
   - `SessionDeviceList.tsx` - 设备列表组件
   - `TerminateConfirmDialog.tsx` - 终止确认弹窗

3. **User-Agent 解析**
   - 使用 `ua-parser-js` 库解析设备信息
   - 提取：浏览器名称、操作系统

### 4.3 通知内容修改

修改 `app/server/src/modules/auth/service.ts` 中的 `notifySessionTerminated` 方法。

## 6. 验收标准

1. 用户登录后，能在设置 → 个人资料页面看到"安全设置"区域
2. 设备列表显示当前所有活跃会话
3. 当前设备有明确标记
4. 能单独终止其他设备
5. 能一键终止所有其他设备
6. 终止操作有二次确认
7. 通知内容清晰：显示新登录设备的 IP 地址
