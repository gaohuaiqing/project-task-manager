# 能力模型管理模块设计规格书

> **版本**: 1.0
> **创建日期**: 2026-03-15
> **状态**: 待审核

---

## 1. 概述

### 1.1 功能定位

能力模型管理模块是任务管理系统的核心功能之一，用于：
- 定义和管理能力评价模型
- 为成员评定能力等级
- 在任务分配时提供智能推荐
- 在报表中展示团队能力分布

### 1.2 设计原则

- **KISS**: 采用两级简化结构，避免过度复杂
- **YAGNI**: 仅实现当前需要的功能，不做过度设计
- **DRY**: 能力模型可复用，成员可关联多个模型

---

## 2. 核心决策汇总

| # | 决策点 | 最终选择 | 理由 |
|---|--------|----------|------|
| 1 | 配置入口位置 | 分离式 | 模型配置在设置中，成员评估在成员管理中 |
| 2 | 层级结构 | 两级结构 | 模型→维度，保持简单便于快速迭代 |
| 3 | 成员关联模型 | 可以多次 | 一个人可同时做嵌入式和质量管理等工作 |
| 4 | 维度字段设计 | 简化版 | 名称、权重(%)、0-100分 |
| 5 | 任务分配建议 | 手动触发 | 点击"智能推荐"按钮显示推荐列表 |
| 6 | 报表能力展示 | 分维度展示 | 列表展示各维度得分 |
| 7 | 雷达图交互 | 点击展开 | 点击成员节点后展开显示雷达图 |

## 3. 数据模型设计

### 3.1 能力模型表 (capability_models)

```typescript
interface CapabilityModel {
  id: string;                    // 主键
  name: string;                  // 模型名称，如"嵌入式开发能力"
  description?: string;          // 模型描述（可选）
  dimensions: Dimension[];       // 评价维度列表
  created_at: Date;
  updated_at: Date;
  created_by: string;            // 创建人ID
}

interface Dimension {
  id: string;                    // 维度ID
  name: string;                  // 维度名称，如"固件开发"
  weight: number;                // 权重百分比，如30表示30%
  // 约束：所有维度权重之和必须为100%
}
```

### 3.2 成员能力评定表 (member_capabilities)

```typescript
interface MemberCapability {
  id: string;                    // 主键
  user_id: string;               // 成员ID
  model_id: string;              // 关联的能力模型ID
  model_name: string;            // 冗余字段，便于查询
  dimension_scores: DimensionScore[]; // 各维度得分
  overall_score: number;         // 加权平均分（计算得出）
  evaluated_at: Date;            // 评定时间
  evaluated_by: string;          // 评定人ID
  notes?: string;                // 备注
}

interface DimensionScore {
  dimension_id: string;          // 维度ID
  dimension_name: string;        // 维度名称
  score: number;                 // 得分（0-100）
}
```

### 3.3 数据约束

1. **权重约束**: 同一模型下所有维度权重之和必须等于100%
2. **评分范围**: 每个维度得分范围为0-100
3. **成员关联**: 同一成员可以关联同一模型多次（不同工作方向）

## 4. 功能模块设计

### 4.1 模块A：能力模型配置（设置模块）

**入口位置**: 设置 → 系统配置 → 能力模型

#### 4.1.1 模型列表页

**UI规格**:
- 使用表格视图展示所有能力模型
- 列：模型名称、维度数量、创建时间、操作
- 操作按钮：编辑、删除

**交互**:
- 点击"新增模型"打开编辑对话框
- 点击"编辑"打开编辑对话框
- 点击"删除"弹出确认对话框

#### 4.1.2 模型编辑对话框

**表单字段**:
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 模型名称 | 文本输入 | 是 | 最大50字符 |
| 模型描述 | 多行文本 | 否 | 最大200字符 |
| 评价维度 | 动态列表 | 是 | 至少1个维度 |

**维度编辑**:
- 维度名称：文本输入，最大30字符
- 维度权重：数字输入，0-100，显示为百分比
- 实时显示权重总和，不为100%时显示警告
- 支持添加/删除维度

### 4.2 模块B：成员能力评定（成员管理模块）

**入口位置**: 成员管理 → 选择成员 → 能力评定Tab

#### 4.2.1 成员能力列表

**UI规格**:
- 使用卡片网格展示成员已关联的能力模型
- 每张卡片显示：模型名称、各维度得分（星级+分数）、综合分数
- 操作按钮：编辑评定、取消关联

#### 4.2.2 能力评定对话框

**表单字段**:
| 字段 | 类型 | 说明 |
|------|------|------|
| 选择能力模型 | 下拉选择 | 从已有模型中选择 |
| 维度评分 | 滑块+数字 | 0-100分，每个维度一个 |

**计算逻辑**:
```typescript
// 综合分数 = 各维度得分 × 权重 之和
overall_score = Σ(dimension_score × weight / 100)
```

### 4.3 模块C：任务分配智能推荐

**触发方式**: 手动点击"智能推荐"按钮

#### 4.3.1 推荐逻辑

```typescript
interface RecommendationResult {
  user_id: string;
  user_name: string;
  match_score: number;           // 匹配度百分比
  capability_details: {
    model_name: string;
    overall_score: number;
    dimension_scores: DimensionScore[];
  }[];
}

// 匹配度计算：
// 1. 根据任务类型匹配相关能力模型
// 2. 计算成员在该模型下的综合分数
// 3. 按分数降序排列
```

#### 4.3.2 UI展示

- 点击"智能推荐"按钮后，在负责人下拉框下方展开推荐面板
- 推荐列表按匹配度降序排列
- 每项显示：成员名称、匹配度分数、星级
- 点击成员可直接选择

### 4.4 模块D：报表能力展示

#### 4.4.1 成员任务分析报表

**能力展示格式**:
```
张三
├── 嵌入式开发能力: 固件开发:85 | 驱动开发:78 | 系统设计:88
└── 质量管理能力: 流程规范:90 | 问题分析:85
```

**UI规格**:
- 使用分维度列表展示
- 每个维度显示：维度名:分数
- 维度间用 `|` 分隔

### 4.5 模块E：组织架构树雷达图

#### 4.5.1 交互方式

- 默认状态：成员节点只显示基本信息（头像、姓名）
- 点击成员节点：展开显示能力雷达图
- 再次点击：收起雷达图

#### 4.5.2 雷达图规格

- 使用三维度雷达图（取综合分数最高的3个维度）
- 如果维度超过3个，显示前3个权重最高的
- 鼠标悬停显示具体分数

## 5. API 设计

### 5.1 能力模型管理 API

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/capability-models` | 获取所有能力模型 |
| GET | `/api/capability-models/:id` | 获取单个能力模型 |
| POST | `/api/capability-models` | 创建能力模型 |
| PUT | `/api/capability-models/:id` | 更新能力模型 |
| DELETE | `/api/capability-models/:id` | 删除能力模型 |

### 5.2 成员能力评定 API

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/members/:id/capabilities` | 获取成员的所有能力评定 |
| POST | `/api/members/:id/capabilities` | 添加成员能力评定 |
| PUT | `/api/members/:id/capabilities/:capId` | 更新成员能力评定 |
| DELETE | `/api/members/:id/capabilities/:capId` | 删除成员能力评定 |

### 5.3 智能推荐 API

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/tasks/recommend-assignee` | 获取任务负责人推荐列表 |

**请求参数**:
```typescript
{
  task_type: string;      // 任务类型
  project_id?: string;    // 项目ID（可选，用于筛选项目成员）
}
```

**响应**:
```typescript
{
  recommendations: RecommendationResult[];
}
```

## 8. 错误处理

### 8.1 业务规则校验

| 场景 | 错误信息 | 处理方式 |
|------|----------|----------|
| 权重之和不为100% | "所有维度权重之和必须等于100%" | 阻止保存，显示提示 |
| 维度名称重复 | "维度名称不能重复" | 阻止保存 |
| 删除已被使用的模型 | "该模型已被使用，无法删除" | 阻止删除 |
| 评分超出范围 | "评分必须在0-100之间" | 自动修正到边界值 |
