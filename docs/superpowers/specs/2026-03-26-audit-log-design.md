# 系统审计日志设计文档

> **创建时间**: 2026-03-26
> **状态**: ✅ 已确认
> **设计目标**: 安全审计 + 业务追溯

---

## 1. 设计决策

| 项目 | 决策 |
|------|------|
| **目的** | 安全审计 + 业务追溯 |
| **变更粒度** | 字段级（old → new） |
| **记录方式** | 关键操作显式调用 |
| **数据结构** | 单表 + category 分类 |
| **页面展示** | 一行一条，筛选器在顶部 |
| **权限** | admin / dept_manager 可见 |
| **性能** | 异步写入 + 索引 + 分页 |
| **保留期限** | 3 个月（自动归档/清理） |

---

## 2. 数据结构

### 2.1 audit_logs 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | UUID 主键 |
| user_id | INT | 操作用户 ID |
| user_name | VARCHAR(50) | 操作用户姓名（冗余，方便查询） |
| category | VARCHAR(20) | 分类（security/project/task/org/config） |
| action | VARCHAR(20) | 操作类型（LOGIN/CREATE/UPDATE/DELETE/ASSIGN...） |
| table_name | VARCHAR(50) | 操作表名（projects/wbs_tasks/users...） |
| record_id | VARCHAR(50) | 记录 ID |
| old_value | JSON | 变更前值（仅 UPDATE 时有值） |
| new_value | JSON | 变更后值（仅 UPDATE 时有值） |
| details | TEXT | 操作描述（如"创建任务：实现登录功能"） |
| ip_address | VARCHAR(45) | 操作 IP |
| user_agent | VARCHAR(255) | 浏览器信息 |
| created_at | DATETIME | 操作时间 |

### 2.2 操作类型枚举

```typescript
// 类别
type AuditCategory = 'security' | 'project' | 'task' | 'org' | 'config';

// 操作类型
type AuditAction =
  // 安全
  | 'LOGIN' | 'LOGOUT' | 'PASSWORD_CHANGE' | 'ROLE_CHANGE'
  // 数据
  | 'CREATE' | 'UPDATE' | 'DELETE' | 'ASSIGN' | 'ARCHIVE' | 'RESTORE'
  // 审批
  | 'APPROVE' | 'REJECT';
```

### 2.3 索引设计

```
主索引: (category, created_at DESC)  -- 分类 + 时间查询
辅助索引: (user_id, created_at DESC) -- 用户查询
```

## 3. 页面设计

### 3.1 筛选区

```
┌─────────────────────────────────────────────────────────────────┐
│ [全部 ▼] [今天 ▼] [选择用户 ▼] [搜索操作内容...]    [导出 Excel] │
└─────────────────────────────────────────────────────────────────┘
```

| 筛选器 | 选项 |
|--------|------|
| 分类 | 全部 / 安全 / 项目 / 任务 / 组织 / 配置 |
| 时间 | 今天 / 近7天 / 近30天 / 自定义范围 |
| 用户 | 全部用户 / 下拉选择 |
| 搜索 | 输入框，匹配操作描述 |

### 3.2 日志列表

```
┌─────────────────────────────────────────────────────────────────┐
│ 2026-03-26 14:30  张三  更新任务  任务#T-001 状态改为"进行中"    │
├─────────────────────────────────────────────────────────────────┤
│ 2026-03-26 14:25  李四  创建项目  新建"Q2营销活动"项目          │
├─────────────────────────────────────────────────────────────────┤
│ 2026-03-26 14:20  王五  登录系统  从 192.168.1.101 登录成功     │
├─────────────────────────────────────────────────────────────────┤
│ 2026-03-26 13:50  张三  更新成员  将李四加入技术组              │
└─────────────────────────────────────────────────────────────────┘
```

**展示规则**：
- 格式：`时间  用户  操作类型  描述内容`
- 颜色区分：创建=绿、更新=蓝、删除=红、登录=紫
- 鼠标悬停显示 IP 地址

## 4. 权限控制

| 角色 | 可见范围 |
|------|----------|
| admin | 所有日志 |
| dept_manager | 所有日志（部门管理需要） |
| tech_manager | ❌ 无权限 |
| engineer | ❌ 无权限 |

---

## 5. 记录触发点

### 5.1 需记录的操作

| 类别 | 记录的操作 | 触发时机 |
|------|-----------|----------|
| **安全** | 登录成功/失败、登出、密码修改 | 认证模块 |
| **安全** | 角色变更、权限调整 | 用户管理模块 |
| **项目** | 创建、更新、删除、归档 | 项目模块 |
| **任务** | 创建、更新、删除、分配、状态变更 | 任务模块 |
| **组织** | 部门/成员的增删改、能力评定 | 组织模块 |
| **配置** | 任务类型/节假日/能力模型配置变更 | 配置模块 |

### 5.2 不记录的操作

| 类型 | 原因 |
|------|------|
| 查询操作（GET） | 只记录变更，不记录查看 |
| 列表翻页 | 无审计价值 |

## 6. 性能指标

| 指标 | 目标 |
|------|------|
| 日志写入延迟 | < 10ms（异步写入，不阻塞业务） |
| 列表查询 | < 200ms（有索引 + 分页） |
| 详情查询 | < 50ms |
| 导出 1000 条 | < 3s |

| 策略 | 说明 |
|------|------|
