## 1. 系统定位

### 1.1 业务定位

| 属性 | 值 |
|------|-----|
| **平台类型** | 企业级任务管理系统 |
| **目标用户** | 100人团队 |
| **部署方式** | 公司内网部署 |
| **设备支持** | 仅桌面端 |

### 1.2 核心价值主张

1. **多项目并行管理** - 同时管理多个项目，支持快速切换
2. **时间驱动规划** - 可视化时间线、甘特图、里程碑管理
3. **数据驱动决策** - 丰富的数据可视化和分析报表
4. **实时协作** - 多人同时编辑，数据自动同步
5. **企业级安全** - 细粒度权限、审计日志、版本控制
6. **能力匹配** - 基于技能模型的智能任务分配

### 1.3 性能目标

| 指标 | 要求 |
|------|------|
| 并发支持 | 100人同时在线 |
| 响应速度 | 加载和操作 < 300ms |
| 任务量 | 单项目100任务不卡顿 |
| 实时性 | 数据同步延迟 < 300ms |
| 冲突处理 | 乐观锁版本控制 |

## 3. 分层设计

### 3.1 表现层（前端）

**职责**：用户界面渲染、用户交互处理、数据展示

#### 组件结构

```
app/src/
├── components/              # UI组件
│   ├── ui/                  # 基础UI组件（shadcn/ui）
│   ├── projects/            # 项目管理组件
│   ├── task-management/     # 任务管理组件
│   └── shared/              # 共享组件
├── hooks/                   # 自定义Hooks
├── services/                # API服务
├── utils/                   # 工具函数
```

#### 状态管理

- **React Query**: 服务器状态管理、数据缓存
- **React State**: 组件本地状态
- **URL State**: 路由参数状态

**职责**：业务逻辑处理、数据验证、权限检查

#### 路由层

```
app/server/src/routes/
├── authRoutes.ts           # 认证路由
├── projectRoutes.ts        # 项目路由
├── taskRoutes.ts           # 任务路由
├── memberRoutes.ts         # 成员路由
├── dataRoutes.ts           # 数据路由
└── ...
```

#### 服务层

```
app/server/src/services/
├── DatabaseService.ts      # 数据库服务
├── GlobalDataManager.ts    # 全局数据管理
├── CacheService.ts         # 缓存服务
└── ...
```

**职责**：数据持久化、缓存管理、数据访问

## 5. 数据流设计

### 5.1 请求数据流

```
用户操作 → React组件 → React Query → API请求
    ↓
Express路由 → 中间件(认证/权限) → 服务层
    ↓
数据验证 → 数据访问层 → 数据库
    ↓
缓存更新 → 响应返回 → React Query缓存更新 → UI更新
```

### 5.2 实时数据流

```
数据变更 → 服务层 → WebSocket服务
    ↓
消息发布 → 订阅客户端 → React Query缓存失效
    ↓
重新获取数据 → UI更新
```

## 6. 安全架构

### 6.1 认证机制

| 机制 | 说明 |
|------|------|
| 会话认证 | Cookie + Redis会话存储（**注意：不使用JWT**） |
| 密码加密 | bcrypt 10轮加盐 |
| 登录限流 | 15分钟5次，锁定30分钟 |
| 多设备控制 | 最多10设备/用户 |
| CSRF防护 | 双重提交Cookie模式 |

**认证流程**:
1. 用户登录 → 验证用户名密码（bcrypt）
2. 创建会话 → 存储到 Redis（24小时TTL） + MySQL（持久化）
3. 设置 Cookie → HttpOnly, Secure, SameSite=Strict
4. 后续请求 → 通过 Cookie 中的 sessionId 验证

**重要配置**:
```env
# 会话配置（参见 app/server/.env.example）
SESSION_SECRET=your_session_secret_here
COOKIE_MAX_AGE=604800000  # 7天
SESSION_MAX_SESSIONS_PER_USER=10
```

### 6.2 权限控制

| 机制 | 说明 |
|------|------|
| 角色权限 | 4种角色，16种细粒度权限 |
| 数据隔离 | 基于项目成员关系 |
| 动态配置 | 运行时权限调整 |

### 6.3 数据安全

| 机制 | 说明 |
|------|------|
| 乐观锁 | 版本控制防冲突 |
| 审计日志 | 完整操作追踪 |
| 数据验证 | 输入校验 |
