## 1. 模块定位

### 仪表板 (07a)
- **业务定位**: 用户登录后第一眼看到的页面
- **核心价值**: 快速发现需要关注的核心问题
- **使用频率**: 每天多次，打开系统即进入
- **用户心态**: 被动接收，扫视状态
- **信息层次**: 状态概览 → 异常信号 → 快速跳转

### 报表分析 (07b)
- **业务定位**: 数据分析与呈现工具
- **核心价值**: 带着问题来，找到答案走
- **使用频率**: 按需，周报/月报/遇到问题时
- **用户心态**: 主动查询，有明确目的
- **信息层次**: 筛选条件 → 统计摘要 → 明细数据 → 导出

### 系统管理 (07c)
- **业务定位**: 系统配置和数据管理
- **核心功能**: 配置管理 + 导入导出

**模块关系**: 仪表板和报表分析**独立使用**，不强制跳转关联，共享数据计算逻辑。

## 3. 核心指标清单（统一计算公式）

| 指标 | 计算公式 | 用途 |
|------|---------|------|
| 完成率 | `已完成任务数 / 总任务数 × 100%` | 仪表板统计卡片 + 报表统计卡片 |
| 延期率 | `延期任务数 / 总任务数 × 100%` | 仪表板统计卡片 + 报表统计卡片 |
| 活跃度 | `7日内有进展的任务数 / 总任务数 × 100%` | 仪表板组效能表格 + 报表资源效能 |
| 负载率 | `进行中任务的全职比总和 / 成员数` | 仪表板成员状态 + 报表成员任务分析 |
| 一周即将延期数 | `count(未来7天到期且进度<100%)` | 仪表板预警卡片 + 任务统计报表 |
| 预估准确性 | `1 - abs(计划工期 - 实际工期) / 计划工期` | 资源效能分析报表 |
| 产能 | `已完成任务数 / 总工时` | 资源效能分析报表 |
| 返工率 | `返工任务数 / 已完成任务数 × 100%` | 资源效能分析报表 |

**延期类型定义**:
- **延期预警**: 未来7天到期且进度<100%
- **已延迟**: 当前日期超过计划结束日期且进度<100%
- **超期完成**: 实际完成日期超过计划结束日期

## 4. API 汇总

| GET | `/stats` | 获取首页统计数据 |

| GET | `/urgent-tasks` | 获取紧急任务列表 |

### 报表分析 API (`/api/reports/`)

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/project-progress` | 项目进度报表 |

| GET | `/delay-analysis` | 延期分析报表 |
| GET | `/member-analysis` | 成员任务分析 |
| GET | `/resource-efficiency` | 资源效能分析报表 |
| GET | `/:type/export` | 导出Excel |

| 方法 | 端点 | 说明 |
|------|------|------|
| GET/POST | `/project-types` | 项目类型列表 |
| GET/POST | `/task-types` | 任务类型列表 |

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/import/projects` | 导入项目数据 |

| GET | `/export/tasks` | 导出任务数据 |

| 趋势指示 | `↑` 绿色 / `↓` 红色 |
| 圆角 | `rounded-lg` (12px) |
| 内边距 | `p-4` |

### 图表规范

| 属性 | 值 |
|------|-----|
| 高度 | `300px` |
| 图表库 | Recharts |
| 动画 | 简洁过渡 |
