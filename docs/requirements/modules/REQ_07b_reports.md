## 0. 业务定位

**定位**：数据分析与呈现工具，用户**带着问题来，找到答案走**。

| 维度 | 说明 |
|------|------|
| 使用频率 | 按需，周报/月报/遇到问题时 |
| 用户心态 | 主动查询，有明确目的 |
| 核心价值 | 按自己的条件分析数据，得出结论 |
| 信息层次 | 筛选条件 → 统计摘要 → 明细数据 → 导出 |

**双重分析维度**：

| 维度类型 | 说明 | 用户价值 |
|---------|------|---------|
| **静态维度** | 当前状态快照 | "现在怎么样" |
| **动态维度** | 时间轴视角的趋势变化 | "怎么变的" |

**动态维度规范**：
- 每个报表都应包含时间轴分析能力
- 支持时间范围选择：过去7天 / 30天 / 本季度 / 自定义
- 展示指标趋势变化线和周期对比（本周 vs 上周）
- **不自动预测**，只呈现趋势数据，由用户自行判断走向

**仪表板与报表的关系**：
```
仪表板（发现异常）→ 点击跳转 → 报表（深入分析原因）
```

## 4. 报表数据维度规格

> **设计原则**：每个报表从5个维度完整定义数据需求，确保前后端数据契约清晰。

### 4.0 核心指标定义

#### 4.0.1 一周即将延迟任务数

| 属性 | 定义 |
|------|------|
| **指标名称** | 一周即将延迟任务数 |
| **业务定义** | 未来7天内到期且未完成的任务数量 |
| **业务含义** | 帮助管理者提前识别即将到期的任务，及时干预避免延期 |
| **显示位置** | 任务统计报表、成员任务分析报表的统计卡片 |

#### 4.0.2 部门成员活跃度

| 属性 | 定义 |
|------|------|
| **指标名称** | 部门成员活跃度 |
| **业务定义** | 部门内7日内有进展更新的任务数占部门总任务数的比例 |
| **计算公式** | `7日内有进展记录的任务数 / 部门总任务数 × 100%` |
| **业务含义** | 100%表示部门所有任务都有最新进展，反映团队工作活跃程度 |
| **显示位置** | 成员任务分析报表统计卡片、资源效能分析报表统计卡片、组效能对比表格、成员效能明细表 |

#### 4.0.3 组效能对比表格指标

| 指标 | 业务定义 | 说明 |
|------|---------|------|
| **完成率** | 已完成任务数占总任务数的比例 | 反映组内任务完成进度 |
| **延期率** | 延期任务数占总任务数的比例 | 反映组内任务延期情况 |
| **负载率** | 进行中任务的全职比总和除以成员数 | 反映组内成员工作负载 |
| **活跃度** | 7日内有进展记录的任务比例 | 反映组内任务更新活跃程度 |
| **成员数** | 时间范围内有任务的成员数量 | 反映组内参与人员规模 |
| **趋势** | 本期完成率与上期完成率的变化 | 反映组内进度变化方向 |
| **状态** | 基于多条件判断的健康状态 | 健康/警告/风险，便于快速识别问题组 |

**组状态判断标准**：

| 状态 | 条件 | 图标 |
|------|------|------|
| **健康** | 完成率 ≥ 80% 且 延期率 ≤ 10% 且 活跃度 ≥ 80% | 🟢 |
| **警告** | 完成率 < 80% 或 延期率 > 10% 或 活跃度 < 80% | 🟡 |
| **风险** | 完成率 < 60% 或 延期率 > 20% 或 负载率 > 110% | 🔴 |

#### 4.0.4 人员调配建议阈值

| 成员状态 | 判断条件 | 业务含义 |
|---------|---------|---------|
| **过载** | 负载率 > 部门平均负载率 × 1.2 | 工作量超出正常范围，需要减轻负担或增加人手 |
| **闲置** | 负载率 < 部门平均负载率 × 0.6 | 工作量明显不足，可以承担更多任务 |
| **低活跃** | 活跃度 < 60% | 任务进展更新不及时，需要关注和督促 |

**数据视图说明要求**：
- 显示当前部门平均负载率
- 显示过载阈值和闲置阈值的计算结果
- 说明低活跃阈值为固定值60%

#### 4.0.5 部门延期率

| 属性 | 定义 |
|------|------|
| **指标名称** | 部门延期率 |
| **业务定义** | 部门延期任务数占部门总任务数的比例 |
| **计算公式** | `部门延期任务数 / 部门总任务数 × 100%` |
| **业务含义** | 反映部门整体延期风险水平 |
| **显示位置** | 延期分析报表统计卡片（dept_manager视图） |

#### 4.0.6 一周即将延期数

| 属性 | 定义 |
|------|------|
| **指标名称** | 一周即将延期数 |
| **业务定义** | 未来7天内到期且进度<100%的任务数量 |
| **计算公式** | `count(wbs_tasks where end_date <= DATE_ADD(NOW(), INTERVAL 7 DAY) AND end_date >= NOW() AND progress < 100)` |
| **业务含义** | 帮助管理者提前识别即将延期的任务，及时干预 |
| **显示位置** | 延期分析报表统计卡片（dept_manager视图） |

#### 4.0.7 成员风险等级判断标准

| 风险等级 | 条件 | 图标 |
|---------|------|------|
| **高风险** 🔴 | 延期率 > 30% 或 (延期率 > 20% 且 活跃度 < 60%) | ⚠️ |
| **中风险** 🟡 | 延期率 10%-30% 或 (延期率 > 10% 且 活跃度 < 60%) | ⚡ |
| **低风险** 🟢 | 延期率 < 10% 且 活跃度 ≥ 60% | ✓ |

### 4.1 数据维度框架

每个报表包含以下5个数据维度：

| 维度 | 说明 | 示例 |
|------|------|------|
| **统计指标** | 统计卡片显示的核心KPI | 任务总数、完成率、延期率 |
| **筛选条件** | 用户可调整的查询参数 | 项目、时间范围、负责人 |
| **静态图表** | 当前状态分布的可视化 | 任务状态饼图、优先级柱状图 |
| **动态图表** | 时间趋势变化的可视化 | 进度趋势线、延期趋势线 |
| **表格数据** | 明细数据的结构化展示 | 任务列表、成员效能表 |

### 4.2 项目进度报表 - 数据维度

**数据来源**：projects表 + wbs_tasks表 + milestones表

| 维度 | 数据项 | 计算公式/说明 |
|------|--------|--------------|
| **统计指标** | 总体进度% | `sum(任务进度 × 任务权重) / 总权重 × 100%` |
| | 已完成任务数 | `count(wbs_tasks where status = 'completed')` |
| | 进行中任务数 | `count(wbs_tasks where status = 'in_progress')` |
| | 里程碑完成数 | `count(milestones where completion_percentage = 100)` |
| **筛选条件** | 项目选择 | 单选或全部（必选） |
| | 时间范围 | 过去7天 / 30天 / 本季度 / 自定义 |
| | 项目类型 | 产品开发 / 职能管理 / 物料改代 / 质量处理 |
| **静态图表** | 任务状态分布饼图 | 9种状态的数量占比 |
| | 里程碑完成情况 | 完成/进行中/逾期里程碑数量 |
| **动态图表** | 进度趋势折线图 | 按时间范围展示进度百分比变化 |
| | 进度变化速度图 | 按周粒度展示每周进度增量 |
| **表格数据** | 里程碑列表 | 名称、目标日期、完成百分比、状态、趋势变化 |

### 4.3 任务统计报表 - 数据维度

**数据来源**：wbs_tasks表

| 维度 | 数据项 | 计算公式/说明 |
|------|--------|--------------|
| **统计指标** | 任务总数 | `count(wbs_tasks)` |
| | 平均完成率 | `avg(任务进度)` |
| | 延期率 | `count(延期任务) / 任务总数 × 100%` |
| | 一周即将延迟任务数 | `count(wbs_tasks where end_date <= DATE_ADD(NOW(), INTERVAL 7 DAY) AND end_date >= NOW() AND status NOT IN ('completed'))` |
| **筛选条件** | 项目 | 多选或全部 |
| | 时间范围 | 过去7天 / 30天 / 本季度 / 自定义 |
| | 负责人 | 多选或全部 |
