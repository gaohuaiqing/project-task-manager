# 模块需求：报表分析 - dept_manager角色视图 (07b-dept)

> **模块编号**: 07b-dept
> **模块名称**: 报表分析 - dept_manager（部门经理）角色视图
> **父模块**: [REQ_07b_reports.md](./REQ_07b_reports.md)

---

## 1. 角色定位

**角色**: dept_manager（部门经理）
**核心视角**: 部门汇总 + 组对比 + 人员调配

**数据范围**:
- 本部门项目
- 本部门成员
- 本部门技术组

**默认报表**: 项目进度报表
**默认筛选**: 本部门项目

---

## 2. 报表视图定义

### 2.1 项目进度报表 - dept_manager视图

**默认筛选**：
- 项目：本部门项目
- 时间范围：过去30天

**差异化内容**：
| 组件 | 内容 |
|------|------|
| 统计卡片 | 部门项目总数、部门任务总数、部门完成率、部门延期率 |
| 图表1 | 本部门项目任务状态分布 |
| 图表2 | 本部门项目里程碑完成情况 |
| 动态图表 | 部门进度趋势、各组进度变化对比 |

**默认筛选**：

### 2.3 延期分析报表 - dept_manager视图

**核心视角**：部门延期态势 + 组对比 + 成员维度分析 + 干预决策支持

**默认筛选**：
- 项目：本部门项目
- 时间范围：过去30天
- 延期类型：全部（延期预警/已延迟/超期完成）

#### 统计卡片（4个）

| 卡片 | 指标 | 计算公式 | 趋势对比 |
|------|------|---------|---------|
| 卡片1 | 部门延期任务总数 | 部门内状态为延期预警/已延迟/超期完成的任务数 | vs 上期 |
| 卡片2 | 部门延期率 | 部门延期任务数 / 部门总任务数 × 100% | vs 上期 |
| 卡片3 | 部门延期预警数 | 部门内状态为"延期预警"的任务数 | vs 上期 |
| 卡片4 | 一周即将延期数 | 未来7天到期且进度<100%的任务数 | — |

#### 图表区域

**图表1（静态）- 组延期分布对比柱状图**

| 元素 | 说明 |
|------|------|
| X轴 | 技术组名称 |
| Y轴 | 延期任务数量 |
| 数据系列 | 延期预警（黄色）、已延迟（红色）、超期完成（灰色） |
| 排序 | 按延期总数降序 |
| 交互 | 点击柱子跳转到该组的延期任务列表 |

**图表2（静态）- 延期原因分类统计**

| 元素 | 说明 |
|------|------|
| 类型 | 水平柱状图 |
| Y轴 | 延期原因（需求变更/技术难度/资源不足/依赖阻塞/其他） |
| X轴 | 任务数量 |
| 排序 | 按数量降序 |
| 交互 | 点击原因跳转到该原因的延期任务列表 |

**图表3（成员维度）- 成员延期×负荷分布图**

| 元素 | 说明 |
|------|------|
| 类型 | 散点图（四象限） |
| X轴 | 任务负荷（全职比总和） |
| Y轴 | 延期任务数 |
| 点大小 | 总任务数 |
| 点颜色 | 活跃度（绿=高、黄=中、红=低） |
| 四象限线 | X=部门平均负荷，Y=部门平均延期数 |
| 交互 | 点击点跳转到该成员的延期任务列表 |

**四象限解读**：
| 象限 | 特征 | 管理建议 |
|------|------|---------|
| 右上（高负荷+高延期） | 过载导致延期 | 减轻负担，增加人手 |
| 左上（低负荷+高延期） | 能力或态度问题 | 需要辅导或督促 |
| 右下（高负荷+低延期） | 高效能成员 | 可以承担更多 |
| 左下（低负荷+低延期） | 闲置或新人 | 可以分配更多任务 |

**图表4（成员维度）- 成员活跃度×延期率分布图**

| 元素 | 说明 |
|------|------|
| 类型 | 散点图（四象限） |
| X轴 | 活跃度（7日内有进展更新的任务比例） |
| Y轴 | 延期率（延期任务数/总任务数） |
| 点大小 | 总任务数 |
| 四象限线 | X=60%（活跃度阈值），Y=部门平均延期率 |
| 交互 | 点击点跳转到该成员的延期任务列表 |

**动态图表1 - 各组延期趋势对比**

| 元素 | 说明 |
|------|------|
| 类型 | 多系列折线图 |
| X轴 | 时间（按周粒度） |
| Y轴 | 延期任务数 |
| 数据系列 | 每个技术组一条线 |
| 图例 | 显示组名 |

**动态图表2 - 部门延期收敛/扩散趋势**

| 元素 | 说明 |
|------|------|
| 类型 | 双系列折线图 |
| X轴 | 时间（按周粒度） |
| Y轴 | 任务数量 |
| 数据系列 | 新增延期（红色）、已解决延期（绿色） |
| 判断 | 收敛=绿色线在红色线上方；扩散=红色线在绿色线上方 |

| 列名 | 字段名 | 类型 | 宽度 | 排序 | 说明 |

**成员延期统计表**

| 列名 | 字段名 | 类型 | 宽度 | 排序 | 说明 |
|------|--------|------|------|:----:|------|
| 成员姓名 | member_name | string | 100px | ✅ | 用户姓名 |
| 所属组 | team_name | string | 100px | ✅ | 技术组名称 |
| 上级主管 | supervisor_name | string | 100px | ✅ | 直接上级主管姓名 |
| 总任务数 | total_tasks | number | 80px | ✅ | 分配的任务总数 |
| 延期任务数 | delayed_tasks | number | 80px | ✅ | 延期的任务数 |
| 延期率 | delay_rate | number | 80px | ✅ | 延期任务数/总任务数×100% |
| 任务负荷 | workload | number | 80px | ✅ | 全职比总和 |
| 活跃度 | activity_rate | number | 80px | ✅ | 7日内有进展更新的任务比例 |
| 风险等级 | risk_level | enum | 80px | ✅ | 高/中/低 |
| 操作 | actions | - | 80px | ❌ | 查看详情 |

**成员风险等级判断标准**：

| 风险等级 | 条件 | 图标 |
|---------|------|------|
| **高风险** | 延期率 > 30% 或 (延期率 > 20% 且 活跃度 < 60%) | ⚠️ |
| **中风险** | 延期率 10%-30% 或 (延期率 > 10% 且 活跃度 < 60%) | ⚡ |
| **低风险** | 延期率 < 10% 且 活跃度 ≥ 60% | ✓ |

#### 特有功能

**1. 组内人员调配建议**

| 建议类型 | 判断条件 | 建议内容 |
|---------|---------|---------|
| 高延期组 | 组延期率 > 部门平均延期率 × 1.5 | 建议增加人员或减少任务分配 |
| 低延期组 | 组延期率 < 部门平均延期率 × 0.5 | 可承担更多任务 |
| 高延期成员 | 成员延期任务数 > 组平均 × 1.5 | 建议减轻负担或提供支持 |

**2. 成员维度调配建议**

| 建议类型 | 判断条件 | 建议内容 |
|---------|---------|---------|
| **过载延期** | 负荷 > 部门平均 × 1.2 且 延期率 > 部门平均 | 减轻负担，分配任务给其他人 |
| **低效延期** | 负荷 < 部门平均 × 0.8 且 延期率 > 部门平均 | 需要辅导，分析延期原因 |
| **低活跃延期** | 活跃度 < 60% 且 延期率 > 部门平均 | 需要督促，了解工作状态 |
| **高效可承担** | 延期率 < 部门平均 × 0.5 且 负荷 < 部门平均 | 可以分配更多任务 |

**3. 延期预警推送设置**

| 功能 | 说明 |
|------|------|
| 自动推送 | 每日推送"一周即将延期"任务列表 |
| 推送渠道 | 系统通知 + 邮件（可选） |
| 推送对象 | 部门经理 + 相关技术经理 |

### 2.4 成员任务分析 - dept_manager视图

**默认筛选**：
- 成员：本部门成员（对比视图）
- 时间范围：过去30天

**差异化内容**：
| 组件 | 内容 |
|------|------|
| 统计卡片 | 部门平均任务负载、部门平均全职比、部门成员活跃度、部门能力匹配度 |
| 图表1 | 组成员负载分布 |
| 图表2 | 组效能对比（含活跃度指标） |
| 动态图表 | 各组负载变化趋势、各组活跃度变化趋势 |
| 数据表格 | 本部门成员任务明细，支持按组筛选 |
| 特有功能 | 人员调配建议（识别过载成员、闲置成员、低活跃成员） |

**人员调配建议阈值说明**：
- 过载阈值 = 部门平均负载率 × 1.2
- 闲置阈值 = 部门平均负载率 × 0.6
- 低活跃阈值 = 活跃度 < 60%（固定值）
