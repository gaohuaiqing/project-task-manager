## 1. WBS任务管理

### 1.1 功能概述

**来源**: FINAL_REQUIREMENTS_0404-0827#L338-346

支持WBS（工作分解结构）任务管理，包括24列复杂表格（24个数据列 + 操作列 = 25列）、任务依赖关系、计划变更审批等功能。这是系统的核心模块，用于分解和管理项目任务。

**优先级**: P0
**实现状态**: 已完整实现

### 1.7 用户场景

**来源**: FINAL_REQUIREMENTS_0404-0827#L500-555

#### 场景1：创建新任务

**前置条件**:
- 已登录系统
- 具有创建任务权限（系统管理员、部门经理、技术经理）
- 已选择或创建项目

**操作步骤**:
1. 进入项目详情页
2. 点击"添加任务"按钮
3. 填写任务基本信息（WBS等级、任务描述）
4. 设置时间信息（开始日期、工期）
5. 选择负责人、优先级等
6. 点击"保存"按钮

**预期结果**:
- 新任务添加到任务列表
- WBS编码自动生成
- 任务状态默认为"未开始"
- 系统记录操作日志

#### 场景2：编辑任务计划

**前置条件**:
- 任务已存在
- 具有编辑权限
- 任务未进入审批流程

**操作步骤**:
1. 点击任务行的"编辑"按钮
2. 修改需要变更的字段
3. 点击"保存"按钮提交变更

**预期结果**:
- 如果是工程师编辑计划字段（开始日期、结束日期、工期、前置任务、提前/落后天数）-> 进入审批流程
- 如果是工程师编辑非计划字段 -> 直接生效
- 如果是技术经理/部门经理/系统管理员 -> 直接生效
- 系统记录变更历史
- 相关人员收到通知

#### 场景3：查看任务历史

**前置条件**:
- 任务已存在
- 任务有历史记录

**操作步骤**:
1. 点击任务行的"进展记录"列
2. 系统弹出历史记录面板
3. 查看所有更新记录

**预期结果**:
- 显示按时间倒序排列的记录列表
- 每条记录包含：时间、更新人、变更内容
- 支持滚动查看所有历史

### 1.8 交互说明

**来源**: FINAL_REQUIREMENTS_0404-0827#L557-622

#### 交互1：添加任务

**触发条件**:
- 点击"添加根任务"按钮
- 点击任务行的"添加子任务"按钮

**操作步骤**:
1. 点击添加按钮
2. 如果是添加子任务，系统自动设置父级关系
3. 表格进入编辑模式，新任务行高亮显示
4. 填写必要信息
5. 点击"保存"按钮或按回车键提交

**系统响应**:
- 保存成功后，新任务添加到列表
- WBS编码自动生成
- 显示成功提示
- 如果验证失败，显示错误信息并停留在编辑模式

**交互反馈**:
- 成功：绿色提示"任务创建成功"
- 失败：红色提示包含具体错误信息

#### 交互2：编辑任务

**触发条件**:
- 点击任务行的"编辑"按钮

**操作步骤**:
1. 点击编辑按钮，任务行进入编辑模式
2. 可编辑字段变为输入框/下拉框
3. 修改需要变更的字段
4. 点击"保存"提交或"取消"放弃修改

**系统响应**:
- 根据用户角色决定是否进入审批流程
- 工程师修改 -> 提交审批
- 技术经理修改 -> 直接保存
- 显示相应的成功提示

**交互反馈**:
- 进入审批：蓝色提示"已提交审批"
- 直接保存：绿色提示"保存成功"

#### 交互3：删除任务

**触发条件**:
- 点击任务行的"删除"按钮

**操作步骤**:
1. 点击删除按钮
2. 系统弹出确认对话框
3. 对话框显示：确认删除任务及其所有子任务？
4. 确认后执行删除

**系统响应**:
- 删除任务及其所有子任务
- 更新其他任务的WBS编码（如果需要）
- 显示删除成功提示

**交互反馈**:
- 确认对话框：警告色（黄色/橙色）
- 删除成功：绿色提示

### 1.10 业务规则

**来源**: FINAL_REQUIREMENTS_0404-0827#L665-804

#### 规则1：WBS编码自动生成

**规则说明**:
- WBS编码由系统自动生成，不可手动编辑
- 编码格式：`父级编码.当前序号`
- 一级任务编码格式：`1`、`2`、`3`...
- 二级任务编码格式：`1.1`、`1.2`、`1.3`...
- 三级任务编码格式：`1.1.1`、`1.1.2`、`1.1.3`...

**应用场景**:
- 创建新任务时自动生成
- 删除任务时不重新编号（保持历史）
- 移动任务时重新计算编码

**示例**:
```
任务1（编码：1）
  ├── 任务1.1（编码：1.1）
  │   └── 任务1.1.1（编码：1.1.1）
  └── 任务1.2（编码：1.2）
任务2（编码：2）
```

#### 规则2：开始日期与前置任务的关系

**规则说明**:
- 当任务有前置任务时，开始日期由系统自动计算
- 计算公式：`开始日期 = 前置任务结束日期 + 提前/落后天数`
- 有前置任务时，开始日期显示为计算后的日期且不可编辑
- 删除前置任务后，开始日期恢复可编辑

**应用场景**:
- 设置前置任务后自动调整开始日期
- 修改前置任务的结束日期时，后续任务自动调整
- 清除前置任务关系后可手动设置日期

**示例**:
```
前置任务：2024-01-01 结束
提前/落后：2天
当前任务开始日期自动计算为：2024-01-03
```

#### 规则3：结束日期计算

**规则说明**:
- 当工期 > 0 时，结束日期由系统自动计算
- 计算时需考虑节假日（从系统配置读取）
- 计算公式：`结束日期 = 开始日期 + 工期 - 1`
- 有工期时，结束日期根据开始日期自动计算完成日期
- 工期为0或空时，可手动设置结束日期

**应用场景**:
- 设置工期后自动计算结束日期
- 修改开始日期时，结束日期自动调整
- 勾选"单休日"时按每周6天计算
- 鼠标选中勾选框提示"勾选表示每周工作6天"

**示例**:
```
开始日期：2024-01-01（周一）
工期：5天（双休日）
结束日期：2024-01-05（周五）

开始日期：2024-01-01（周一）
工期：5天（单休日）
结束日期：2024-01-06（周六）
```

#### 规则4：任务状态自动判断

**判断逻辑**（依次判断1-9，第一个满足条件的就是最终状态）:

| 序号 | 状态 | 判断条件 |
|------|------|---------|
| 1 | 待审批 | 上级主管未处理 |
| 2 | 已驳回 | 上级主管审批未通过 |
| 3 | 未开始 | 没有完整计划日期或有计划日期但无实际开始日期 |
| 4 | 进行中 | 有计划日期、实际开始日期，无实际完成日期且未超期 |
| 5 | 提前完成 | 有实际完成日期且早于计划完成日期 |
| 6 | 按时完成 | 有实际完成日期且等于计划完成日期 |
| 7 | 延期预警 | 无实际完成日期且当前距离计划完成日期 <= 预警天数 |
| 8 | 已延迟 | 无实际完成日期且当前已超过计划完成日期 |
| 9 | 超期完成 | 有实际完成日期且晚于计划完成日期 |

#### 规则5：计划周期计算

**规则说明**:
- 计划周期由系统自动计算，不可编辑
- 计算公式：`计划周期 = 结束日期 - 开始日期 + 1`
- 单位：天（日历天数）
- 显示在"计划周期"列

**应用场景**:
- 修改开始或结束日期时自动更新
- 用于统计分析项目周期

#### 规则5.5：实际工期计算

**规则说明**:
- 实际工期由系统自动计算，不可编辑
- 计算公式：统计实际开始日期到实际结束日期之间的**工作日数**（跳过周末和节假日）
- 单位：天（工作日）
- 只有当实际开始和实际结束都有值时才计算

**应用场景**:
- 任务完成后统计实际投入的工作时间
- 用于工作量分析和效率评估
- 与实际周期对比可分析非工作日影响

#### 规则6：实际周期计算

**规则说明**:
- 实际周期由系统自动计算，不可编辑
- 计算公式：`实际周期 = 实际结束日期 - 实际开始日期 + 1`
- 单位：天（日历天数，含周末和节假日）
- 只有当实际开始和实际结束都有值时才计算

**与实际工期的区别**:
- 实际周期 = 日历天数（含周末节假日）
- 实际工期 = 工作日数（跳过周末节假日）

#### 规则7：全职比影响

**规则说明**:
- 全职比表示任务投入的人力资源比例
- 取值范围：0-100
- 100表示全职，50表示半职
- 用于资源统计和负荷分析

#### 规则8：工程师任务权限详细说明

**适用角色**: 工程师（engineer）

**权限矩阵**:

| 操作 | 权限 | 审批要求 |

**创建子任务规则**:

|------|------|
