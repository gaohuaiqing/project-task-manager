> **模块编号**: 07a
> **模块名称**: 仪表板（数据看板）
> **父模块**: [REQ_07_analytics.md](./REQ_07_analytics.md)

## 0. 业务定位

**定位**：用户登录后第一眼看到的页面，目标是**快速发现需要关注的核心问题**。

| 维度 | 说明 |
|------|------|
| 使用频率 | 每天多次，打开系统即进入 |
| 用户心态 | 被动接收，扫视状态 |
| 核心价值 | 让用户不用翻页就知道有没有问题 |
| 信息层次 | 状态概览 → 异常信号 → 快速跳转 |

**角色差异化内容**：

| 角色 | 关注范围 | 核心信息 |
|------|---------|---------|
| admin | 全局 | 全部项目健康状态、团队负载概况、延期预警 |
| dept_manager | 本部门 | 部门项目进度、部门人员任务状态 |
| tech_manager | 本技术组 + 被授权技术组 | 组内任务进展、成员负载 |
| engineer | 自己参与的项目 | 项目进展、个人待办和到期任务 |

**设计原则**：
- 只放用户需要**立刻关注**的信息
- 不放导航入口、通知中心等非数据类内容（这些由侧边栏和独立模块承担）
- 卡片应支持趋势指标（如"↑5% vs 上周"），帮助用户感知变化

## 1. UI风格要求

**设计参考**: https://api.svips.org/dashboard

**核心风格定位**: 专业工具软件风格，高级感、科技感，数据驱动

### 1.1 配色规范

| 元素 | 浅色主题 | 深色主题 |
|------|----------|----------|
| 背景色 | 纯白 `#FFFFFF` | 深蓝黑 `#0A0F1C` |
| 卡片背景 | 白色 `#FFFFFF` | 深蓝灰 `#1A1F2E` |
| 主数值 | 深色 `#0F172A` | 亮白 `#F1F5F9` |
| 标签文字 | 灰色 `#64748B` | 灰色 `#94A3B8` |
| 强调色 | 青蓝 `#0EA5E9` | 青蓝 `#38BDF8` |

### 1.2 统计卡片规范

| 元素 | 规范 |
|------|------|
| 主数值字号 | 28px 加粗 |
| 主数值字体 | 等宽字体 `font-mono`，`tabular-nums` |
| 标签字号 | 12px 中等字重 |
| 卡片圆角 | 12px |
| 卡片边框 | 1px 实线，半透明 |
| 卡片阴影 | 轻微阴影，hover时加深 |
| Hover效果 | 微微上浮 + 阴影加深 |

### 1.3 图表规范

| 元素 | 规范 |
|------|------|
| 图表背景 | 与卡片背景一致 |
| 网格线 | 浅灰色，半透明 |
| 数据线 | 青蓝色为主，绿色为辅 |
| 图例位置 | 图表下方或右侧 |
| 响应式 | 高度固定300px，宽度自适应 |

### 1.4 布局规范

| 元素 | 规范 |
|------|------|
| 统计卡片区 | 4列网格，gap-4 |
| 图表区 | 2列网格，gap-6 |
| 整体间距 | space-y-6 |
| 内边距 | p-4 或 p-6 |

### 1.5 专业感要素

1. **数据层次分明**: 主数值 > 标签 > 辅助信息
2. **等宽数字**: 所有数值使用 `tabular-nums`，避免数字跳动
3. **紧凑但不拥挤**: 信息密度高，间距适中
4. **深浅双主题支持**: 两种主题下都有专业感

## 2. 首页概览

**来源**: FINAL_REQUIREMENTS_0404-0827#L93-94, UI_Requirement_0404-0827#L2988-3009

**功能概述**: 提供首页工作台和仪表盘，快速展示关键业务指标。

### 2.1 统计卡片

| 决策项 | 最终决策 |
|--------|----------|
| 样式 | 极简数字（去除图标） |
| 数量 | 4个卡片 |
| 布局 | grid-cols-4 |

**卡片内容**（按角色差异化）:

| 卡片 | admin/dept_manager/tech_manager | engineer |
|------|--------------------------------|----------|
| 卡片1 | 项目总数（关注范围内） | 参与项目数 |
| 卡片2 | 进行中任务数 | 我的进行中任务数 |
| 卡片3 | 已完成任务数 | 我的已完成任务数 |
| 卡片4 | 延期预警数 | 我的到期/逾期任务数 |

**趋势指标**：

每个卡片右下角显示与上周同期的对比趋势：
- 上升指标（如已完成任务）：绿色 ↑ 表示正向，红色 ↓ 表示负向
- 下降指标（如延期预警）：绿色 ↓ 表示正向，红色 ↑ 表示负向
- 格式示例：`↑ 12% vs 上周` 或 `↓ 3 vs 上周`
- 数据来源：对比当前7天与前7天的同一指标

### 2.2 图表区域

| 决策项 | 最终决策 |
|--------|----------|
| 布局方式 | 多个小图表 |
| 图表类型 | 任务趋势（折线图）、项目进度分布（饼图） |
| 状态 | 需新增 |

### 2.3 紧急任务提醒

| 元素 | 规范 |
|------|------|
| 样式 | 独立警告卡片 |
| 颜色 | 红色边框 + 浅红背景 |
| 内容 | 显示紧急任务数量 + 快速跳转 |

## 3. 仪表板组件

**来源**: UI_Requirement_0404-0827#L2281

### 3.1 组件清单

**核心组件（保留）**:
1. 统计卡片组件 — 4个核心指标，支持趋势指标（如 ↑5% vs 上周）
2. 趋势图表组件 — 30天任务趋势（新建/完成/延期）
3. 项目进度组件 — 当前用户关注范围内的项目列表及进度
4. 紧急任务提醒组件 — 延期预警和逾期任务告警
5. 任务列表组件 — 当前用户的待办任务快览

**辅助组件（角色差异化显示）**:
6. 任务分布组件 — 按状态/优先级/负责人的分布概览（仅 manager 角色显示）
7. 状态饼图组件 — 任务状态分布饼图（仅 manager 角色显示）

> **移除说明**：通知中心组件（NotificationCenter）和快捷操作组件（QuickActions）不属于仪表板核心定位，不再纳入仪表板模块。通知功能由独立通知模块承担，导航入口由侧边栏承担。

### 3.2 角色差异化

> - admin/dept_manager/tech_manager：显示全部7个组件，数据范围按角色过滤
> - engineer：显示统计卡片（个人）、任务列表、紧急任务提醒、项目进度（仅自己参与的项目），不显示分布/饼图类分析组件

**活跃度指标定义** ✨：

| 指标 | 计算公式 | 说明 |
|------|---------|------|
| 部门活跃度 | 部门内7日内有进展更新的任务数 / 部门总任务数 × 100% | 100%表示部门所有任务都有最新进展 |
| 组活跃度 | 组内7日内有进展更新的任务数 / 组总任务数 × 100% | 组级别的活跃度指标 |

**组状态判断标准**：

| 状态 | 条件 | 图标 |
|------|------|------|
| 健康 | 完成率 ≥ 80% 且 延期率 ≤ 10% 且 活跃度 ≥ 80% | 🟢 |
| 警告 | 完成率 < 80% 或 延期率 > 10% 或 活跃度 < 80% | 🟡 |
| 风险 | 完成率 < 60% 或 延期率 > 20% 或 负载率 > 110% | 🔴 |

**任务类型分析维度**：

| 任务类型 | 数据项 |
|---------|-------|
| 固件/板卡/驱动/接口类等 | 任务数量、占比、平均工期、延期率 |

**tech_manager 特有组件**：

| 组件 | 类型 | 说明 |
|------|------|------|

| 组内核心指标卡片 | 8个统计卡片 | 组内项目数、任务数、完成率、延期率、人数、平均负载、本周到期、活跃度 |
| 成员任务状态表格 | 可排序表格 | 横向对比组内成员的任务数量、完成率、负载率、活跃度 |
| 成员任务分布图 | 堆叠柱状图 | 展示每个成员的任务状态分布 |
| 成员活跃度趋势图 ✨ | 多线图 | 展示各成员的7日内更新率变化 |
| 任务分配建议 | 建议卡片 | 智能识别过载成员、可接单成员、低活跃成员 |

**活跃度指标定义** ✨：

| 指标 | 计算公式 | 说明 |
|------|---------|------|
| 成员活跃度 | 7日内有进展更新的任务数 / 总任务数 × 100% | 100%表示所有任务都有最新进展 |

**状态判断标准**：

| 状态 | 条件 | 图标 |
|------|------|------|
| 正常 | 负载率 70%-100% 且 活跃度 ≥ 80% | 🟢 |
| 过载 | 负载率 > 100% | 🟡 |
| 低活跃 | 活跃度 < 80% | 🟡 |
| 空闲 | 负载率 < 70% | 🔵 |
