## 0. 业务定位

### 0.1 仪表板 — 业务定位

**定位**：用户登录后第一眼看到的页面，目标是**快速发现需要关注的核心问题**。

| 维度 | 说明 |
|------|------|
| 使用频率 | 每天多次，打开系统即进入 |
| 用户心态 | 被动接收，扫视状态 |
| 核心价值 | 让用户不用翻页就知道有没有问题 |
| 信息层次 | 状态概览 → 异常信号 → 快速跳转 |

**角色差异化内容**：

| 角色 | 关注范围 | 核心信息 |
|------|---------|---------|
| admin | 全局 | 全部项目健康状态、团队负载概况、延期预警 |
| dept_manager | 本部门 | 部门项目进度、部门人员任务状态 |
| tech_manager | 本技术组 + 被授权技术组 | 组内任务进展、成员负载 |
| engineer | 自己参与的项目 | 项目进展、个人待办和到期任务 |

**设计原则**：
- 只放用户需要**立刻关注**的信息
- 不放导航入口、通知中心等非数据类内容（这些由侧边栏和独立模块承担）
- 卡片应支持趋势指标（如"↑5% vs 上周"），帮助用户感知变化

### 0.2 报表分析 — 业务定位

**定位**：数据分析与呈现工具，用户**带着问题来，找到答案走**。

| 维度 | 说明 |
|------|------|
| 使用频率 | 按需，周报/月报/遇到问题时 |
| 用户心态 | 主动查询，有明确目的 |
| 核心价值 | 按自己的条件分析数据，得出结论 |
| 信息层次 | 筛选条件 → 统计摘要 → 明细数据 → 导出 |

**双重分析维度**：

| 维度类型 | 说明 | 用户价值 |
|---------|------|---------|
| **静态维度** | 当前状态快照 | "现在怎么样" |
| **动态维度** | 时间轴视角的趋势变化 | "怎么变的" |

**动态维度规范**：
- 每个报表都应包含时间轴分析能力
- 支持时间范围选择：过去7天 / 30天 / 本季度 / 自定义
- 展示指标趋势变化线和周期对比（本周 vs 上周）
- **不自动预测**，只呈现趋势数据，由用户自行判断走向

**仪表板与报表的关系**：
```
仪表板（发现异常）→ 点击跳转 → 报表（深入分析原因）
```

### 1.2 仪表板组件

**来源**: UI_Requirement_0404-0827#L2281

**组件清单（5个核心组件 + 2个辅助组件）**:

核心组件（保留）:
1. 统计卡片组件 — 4个核心指标，支持趋势指标（如 ↑5% vs 上周）
2. 趋势图表组件 — 30天任务趋势（新建/完成/延期）
3. 项目进度组件 — 当前用户关注范围内的项目列表及进度
4. 紧急任务提醒组件 — 延期预警和逾期任务告警
5. 任务列表组件 — 当前用户的待办任务快览

辅助组件（角色差异化显示）:
6. 任务分布组件 — 按状态/优先级/负责人的分布概览（仅 manager 角色显示）
7. 状态饼图组件 — 任务状态分布饼图（仅 manager 角色显示）

> **移除说明**：通知中心组件（NotificationCenter）和快捷操作组件（QuickActions）不属于仪表板核心定位，不再纳入仪表板模块。通知功能由独立通知模块承担，导航入口由侧边栏承担。

> **角色差异化**：
> - admin/dept_manager/tech_manager：显示全部7个组件，数据范围按角色过滤
> - engineer：显示统计卡片（个人）、任务列表、紧急任务提醒、项目进度（仅自己参与的项目），不显示分布/饼图类分析组件

#### 预估准确性指标说明（v1.2 新增）

| 指标 | 计算公式 | 说明 |
|------|---------|------|
| 预估准确性 | `1 - |实际工期 - 计划工期| / 计划工期` | 100%表示完全准确 |
| 预估偏差率 | `(实际工期 - 计划工期) / 计划工期 * 100%` | 正数表示低估，负数表示高估 |
| 预估偏差天数 | `实际工期 - 计划工期` | 正数表示延期天数 |

**预估准确性分级**:
| 等级 | 偏差范围 | 标签颜色 |
|------|---------|---------|
| 精准 | ±10% | 🟢 绿色 |
| 轻微偏差 | ±10%~30% | 🟡 黄色 |
| 明显偏差 | ±30%~50% | 🟠 橙色 |
| 严重偏差 | >±50% | 🔴 红色 |

### 2.6 资源效能分析报表 ✨ v1.2 新增

**来源**: 管理视角需求补充，2026-04-04

**优先级**: P1
**实现状态**: ✅ 已实现

**功能概述**: 从资源利用和人员效能角度分析团队生产力，帮助管理者识别高绩效成员、优化资源配置、发现能力短板。

| 元素 | 静态维度（现在怎样） | 动态维度（怎么变的） |
|------|-------------------|-------------------|
| 统计卡片 | 平均产能、平均预估准确性、平均返工率、全职比利用率 | 每个卡片显示效能指标的变化趋势 |
| 图表1 | 人员产能排名柱状图 | 产能变化趋势（按周/月展示） |
| 图表2 | 预估准确性分布图 | 团队效能对比（按部门/技术组） |
| 数据表格 | 成员效能明细（成员、完成任务数、产能、预估准确性、返工率、全职比利用率） | — |
| 导出内容 | 效能统计 + 成员明细 + 趋势数据 |

**筛选条件**: 部门、时间范围、产能阈值
**数据来源**: wbs_tasks表 + users表

#### 核心指标定义

| 指标 | 计算公式 | 业务含义 |
|------|---------|---------|
| **人员产能** | `完成任务数 × 复杂度系数 / 投入天数` | 单位时间的有效产出 |
| **预估准确性** | `1 - |实际工期 - 计划工期| / 计划工期` | 计划能力的准确程度 |
| **返工率** | `计划调整次数 / 任务数 × 100%` | 任务规划质量（低返工率=高质量） |
| **全职比利用率** | `实际投入全职比 / 分配全职比 × 100%` | 资源利用效率 |

#### 任务复杂度系数

| WBS等级 | 复杂度系数 | 说明 |
|:-------:|:---------:|------|
| 1级 | 1.0 | 基准级别 |
| 2级 | 1.2 | 子任务 |
| 3级 | 1.5 | 更细粒度任务 |
| 4级+ | 2.0 | 最细粒度任务 |

#### 报表权限

| 角色 | 可见范围 |
|------|---------|
| admin | 全部成员 |
| dept_manager | 本部门成员 |
| tech_manager | 本技术组 + 被授权技术组成员 |
| engineer | 不可见 |

#### 管理价值

| 分析场景 | 支持决策 |
|---------|---------|
| 高绩效成员识别 | 奖励激励、最佳实践复制 |
| 低效能成员发现 | 针对性辅导、培训需求识别 |
| 资源配置优化 | 识别资源瓶颈、调整任务分配 |
| 团队能力建设 | 技能分布分析、培训计划制定 |
