## 1. 实时协作

### 1.1 功能概述

**来源**: FINAL_REQUIREMENTS_0404-0827#L1206-1215

支持多人实时协作，包括数据自动同步、实时通知、在线状态显示等功能。确保团队成员始终看到最新数据。

### 1.2 实时协作增强功能

| 增强功能 | 描述 | 来源 |

| 消息去重 | 通过nodeId过滤自己发布的消息 | 代码 |
| 跨标签页同步 | 使用BroadcastChannel实现跨浏览器标签页同步 | 代码 |
| 在线状态 | 实时显示用户在线状态 | 代码 |
| 消息优先级 | 支持高优先级消息优先发送 | 代码 |

### 1.3 用户场景

#### 场景1：多人同时查看任务

**来源**: FINAL_REQUIREMENTS_0404-0827#L1227-1242

**前置条件**:
- 多个用户同时在线
- 都打开了任务列表

**操作步骤**:
1. 用户A打开任务列表
2. 用户B打开任务列表
3. 用户C修改了一个任务
4. 用户A和用户B看到更新

**预期结果**:
- 用户A和B实时看到任务变更
- 无需手动刷新页面
- 变更在300ms内同步

#### 场景2：接收实时通知

**来源**: FINAL_REQUIREMENTS_0404-0827#L1244-1258

**前置条件**:
- 您已提交审批请求

**操作步骤**:
1. 工程师提交审批
2. 技术经理审批（通过或驳回）
3. 工程师收到通知

**预期结果**:
- 工程师实时收到审批结果通知
- 通知内容清晰明确
- 可以点击通知跳转到相关页面

#### 场景3：查看团队成员在线状态

**来源**: FINAL_REQUIREMENTS_0404-0827#L1260-1273

**前置条件**:
- 您已登录系统

**操作步骤**:
1. 打开团队成员列表
2. 查看成员在线状态
3. 根据状态决定是否联系

**预期结果**:
- 可以看到团队成员的在线状态
- 帮助判断是否可以即时沟通
- 显示：在线/离开/离线

### 1.4 业务规则

#### 规则1：同步范围控制

**来源**: FINAL_REQUIREMENTS_0404-0827#L1276-1287

**规则说明**:
- 只同步与您相关的数据变更
- 根据您参与的项目过滤同步内容
- 系统管理员可以看到所有项目的变更

#### 规则2：同步延迟要求

**来源**: FINAL_REQUIREMENTS_0404-0827#L1288-1297

**规则说明**:
- 变更在0.3秒（300ms）内自动同步到其他用户
- 从您保存修改到其他人看到更新

#### 规则3：同时编辑处理

**来源**: FINAL_REQUIREMENTS_0404-0827#L1298-1310

**规则说明**:
- 后保存的修改覆盖先保存的修改
- 版本冲突时显示冲突对话框
- 记录覆盖操作

#### 规则4：在线状态管理

**来源**: FINAL_REQUIREMENTS_0404-0827#L1312-1321

**规则说明**:
- 您登录后显示为"在线"
- 您退出后显示为"离线"
- 5分钟无操作显示为"离开"

## 2. 版本控制与冲突解决

### 2.1 功能概述

**来源**: FINAL_REQUIREMENTS_0404-0827#L1630-1641

| 功能 | 描述 |
|------|------|
| 乐观锁机制 | 每次更新检测version字段 |
| 冲突检测 | 版本不匹配时返回409错误 |
| 版本历史 | 记录每次变更的历史版本 |
| 冲突解决 | 提供冲突对话框，用户可选择覆盖或放弃 |

### 2.2 API端点

```
PUT /api/projects/v2/:id    # 带版本控制的项目更新
PUT /api/tasks/v2/:id       # 带版本控制的任务更新
```

### 2.3 版本控制规则

| 规则 | 说明 |
|------|------|
| 版本字段 | 每条记录包含version字段（整数） |
| 更新时检测 | 更新时必须携带当前版本号 |
| 冲突处理 | 版本不匹配返回409错误 |
| 版本递增 | 更新成功后版本号自动+1 |

## 3. 批量操作

### 3.1 功能概述

**来源**: FINAL_REQUIREMENTS_0404-0827#L1648-1660

| 功能 | API端点 | 说明 |

## 4. 缓存管理

### 4.1 功能概述

**来源**: FINAL_REQUIREMENTS_0404-0827#L1680-1699

| 功能 | 描述 |
|------|------|
| Redis主缓存 | 高性能缓存支持 |
| LRU降级 | Redis不可用时自动降级到内存缓存 |
| 缓存预热 | 系统启动时预加载热点数据 |
| 缓存清理 | 支持手动清理和自动过期 |

### 4.2 缓存策略

| 数据类型 | 过期时间 | 降级策略 |
|---------|---------|---------|
| 项目列表 | 5分钟 | LRU缓存 |
| 成员列表 | 10分钟 | LRU缓存 |
| 任务列表 | 2分钟 | LRU缓存 |
| 配置数据 | 30分钟 | 内存缓存 |

## 接口需求

### API列表

#### 实时协作
```
WebSocket /ws              # WebSocket连接
POST /api/events/publish   # 发布事件
GET  /api/online-users     # 获取在线用户
```

#### 批量操作
```
POST /api/batch/projects     # 批量获取项目
POST /api/batch/members      # 批量获取成员
POST /api/batch/wbs-tasks    # 批量获取任务
POST /api/batch/mixed        # 混合批量查询
POST /api/batch/cache/warmup # 缓存预热
```

#### 版本控制
```
PUT /api/projects/v2/:id     # 带版本控制的项目更新
PUT /api/tasks/v2/:id        # 带版本控制的任务更新
GET /api/projects/:id/versions # 获取项目版本历史
GET /api/tasks/:id/versions   # 获取任务版本历史
```

#### 缓存管理
```
DELETE /api/cache/clear      # 清理缓存
POST   /api/cache/warmup     # 缓存预热
GET    /api/cache/status     # 缓存状态
```

#### 附件管理
```
GET    /api/tasks/:id/attachments  # 获取任务附件
POST   /api/tasks/:id/attachments  # 上传附件
DELETE /api/attachments/:id        # 删除附件
```
