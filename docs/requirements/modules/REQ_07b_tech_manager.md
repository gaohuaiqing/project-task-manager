## 1. 角色定位

**角色**: tech_manager（技术经理）
**核心视角**: 组汇总 + 成员对比 + 任务分配

**数据范围**:
- 本技术组参与的项目
- 本技术组成员

**默认报表**: 项目进度报表
**默认筛选**: 本技术组参与的项目

### 2.3 延期分析报表 - tech_manager视图

**核心视角**：组延期态势 + 成员深度分析 + 任务分配优化

**默认筛选**：
- 项目：本技术组参与的项目
- 时间范围：过去30天
- 延期类型：全部（延期预警/已延迟/超期完成）

#### 统计卡片（4个）

| 卡片 | 指标 | 计算公式 | 趋势对比 |
|------|------|---------|---------|
| 卡片1 | 组延期任务总数 | 组内状态为延期预警/已延迟/超期完成的任务数 | vs 上期 |
| 卡片2 | 组延期率 | 组延期任务数 / 组总任务数 × 100% | vs 上期 |
| 卡片3 | 组延期预警数 | 组内状态为"延期预警"的任务数 | vs 上期 |
| 卡片4 | 组超期完成数 | 组内状态为"超期完成"的任务数 | vs 上期 |

#### 图表区域

**图表1（静态）- 成员延期分布柱状图**

| 元素 | 说明 |
|------|------|
| X轴 | 成员姓名 |
| Y轴 | 延期任务数量 |
| 数据系列 | 延期预警（黄色）、已延迟（红色）、超期完成（灰色） |
| 排序 | 按延期总数降序 |
| 交互 | 点击柱子跳转到该成员的延期任务列表 |

**图表2（静态）- 延期任务类型分布**

| 元素 | 说明 |
|------|------|
| 类型 | 水平柱状图 |
| Y轴 | 任务类型（固件/板卡/驱动等12类） |
| X轴 | 延期任务数量 |
| 排序 | 按数量降序 |
| 交互 | 点击类型跳转到该类型的延期任务列表 |

**图表3（成员维度）- 成员延期×负荷分布图**

| 元素 | 说明 |
|------|------|
| 类型 | 散点图（四象限） |
| X轴 | 任务负荷（全职比总和） |
| Y轴 | 延期任务数 |
| 点大小 | 总任务数 |
| 点颜色 | 活跃度（绿=高、黄=中、红=低） |
| 四象限线 | X=组平均负荷，Y=组平均延期数 |
| 交互 | 点击点跳转到该成员的延期任务列表 |

**图表4（成员维度）- 成员活跃度×延期率分布图**

| 元素 | 说明 |
|------|------|
| 类型 | 散点图（四象限） |
| X轴 | 活跃度（7日内有进展更新的任务比例） |
| Y轴 | 延期率（延期任务数/总任务数） |
| 点大小 | 总任务数 |
| 四象限线 | X=60%（活跃度阈值），Y=组平均延期率 |
| 交互 | 点击点跳转到该成员的延期任务列表 |

**四象限解读**：
| 象限 | 特征 | 管理建议 |
|------|------|---------|
| 右上（高活跃+高延期） | 努力但延期 | 可能任务难度大，需要支持 |
| 左上（低活跃+高延期） | 不积极且延期 | 需要重点关注和督促 |
| 右下（高活跃+低延期） | 高效成员 | 榜样，可以承担更多 |
| 左下（低活跃+低延期） | 任务少或新人 | 需要了解情况 |

**动态图表1 - 成员延期趋势对比**

| 元素 | 说明 |
|------|------|
| 类型 | 多系列折线图 |
| X轴 | 时间（按周粒度） |
| Y轴 | 延期任务数 |
| 数据系列 | 每个成员一条线 |
| 图例 | 显示成员名 |

**动态图表2 - 组延期收敛/扩散趋势**

| 元素 | 说明 |
|------|------|
| 类型 | 双系列折线图 |
| X轴 | 时间（按周粒度） |
| Y轴 | 任务数量 |
| 数据系列 | 新增延期（红色）、已解决延期（绿色） |
| 判断 | 收敛=绿色线在红色线上方；扩散=红色线在绿色线上方 |

#### 特有功能

**1. 成员维度任务分配建议**

| 建议类型 | 判断条件 | 建议内容 |
|---------|---------|---------|
| **过载延期** | 负荷 > 组平均 × 1.2 且 延期率 > 组平均 | 减轻负担，分配任务给其他成员 |
| **低效延期** | 负荷 < 组平均 × 0.8 且 延期率 > 组平均 | 需要辅导，分析延期原因 |
| **低活跃延期** | 活跃度 < 60% 且 延期率 > 组平均 | 需要督促，了解工作状态 |
| **高效可接单** | 延期率 < 组平均 × 0.5 且 负荷 < 组平均 | 可以承担更多任务 |

**2. 延期成员一对一跟进**

| 功能 | 说明 |
|------|------|
| 关注名单 | 自动标记高风险成员，方便持续关注 |
| 进展提醒 | 为关注成员设置进展更新提醒 |
| 辅导记录 | 记录与延期成员的沟通和辅导情况 |
