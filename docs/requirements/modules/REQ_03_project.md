## 1. 项目管理

### 1.1 项目CRUD

**来源**: FINAL_REQUIREMENTS#L156-165

| 功能 | 描述 | 优先级 | API端点 |
|------|------|--------|---------|
| 创建项目 | 填写项目信息并创建 | P0 | POST /api/projects |
| 编辑项目 | 修改项目信息（带版本控制） | P0 | PUT /api/projects/v2/:id |
| 删除项目 | 删除项目（需确认） | P0 | DELETE /api/projects/v2/:id |
| 查看项目列表 | 查看所有有权限的项目 | P0 | GET /api/projects |
| 查看项目详情 | 查看项目完整信息 | P0 | GET /api/projects/:id/detail |

**项目字段规格**:

| 字段名称 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| id | UUID | 是 | 主键 |
| code | String(50) | 是 | 项目代号（全局唯一） |
| name | String(100) | 是 | 项目名称 |
| description | Text | 否 | 项目描述 |
| status | Enum | 是 | 状态（planning/active/completed） |
| project_type | Enum | 是 | 类型（见项目类型表） |
| planned_start_date | Date | 是 | 计划开始日期 |
| planned_end_date | Date | 是 | 计划结束日期 |
| actual_start_date | Date | 否 | 实际开始日期 |
| actual_end_date | Date | 否 | 实际结束日期 |
| progress | Integer | 是 | 进度百分比（0-100） |
| task_count | Integer | 是 | 任务总数（自动统计） |
| completed_task_count | Integer | 是 | 已完成任务数（自动统计） |
| member_ids | Text | 否 | 成员ID列表（逗号分隔） |
| version | Integer | 是 | 版本号（乐观锁） |
| created_at | Timestamp | 是 | 创建时间 |
| updated_at | Timestamp | 是 | 更新时间 |

**项目类型默认值**:

| 序号 | 项目类型 | 编码 | 说明 |
|------|---------|------|------|
| 1 | 产品开发 | product_dev | 新产品研发项目 |
| 2 | 职能管理 | func_mgmt | 职能部门管理项目 |
| 3 | 物料改代 | material_sub | 物料替代改进项目 |
| 4 | 质量处理 | quality_handle | 质量问题处理项目 |

**验收标准**:
- 创建项目时必填字段：项目代号、项目名称、项目类型、计划开始/结束日期
- 项目代号全局唯一，不可重复
- 结束日期不能早于开始日期
- 删除项目时需二次确认
- 有任务的项目不能删除

### 1.2 项目成员管理

**来源**: FINAL_REQUIREMENTS#L197-204

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 添加项目成员 | 从组织架构选择成员添加到项目 | P0 |
| 移除项目成员 | 从项目中移除成员 | P0 |
| 设置成员角色 | 设置成员在项目中的角色 | P0 |
| 查看项目成员 | 查看项目成员列表 | P0 |

**API端点**: `GET/POST /api/projects/:id/members`

**验收标准**:
- 只有项目经理和系统管理员可以添加/移除成员
- 成员添加后可立即访问项目数据
- 移除成员后该成员无法访问项目数据

### 1.3 业务规则

**来源**: FINAL_REQUIREMENTS#L286-334

| 规则ID | 规则名称 | 描述 |
|--------|---------|------|
| PRJ-RULE-01 | 项目类型关联 | 项目类型关联任务类型 |
| PRJ-RULE-02 | 项目时间约束 | 结束日期不能早于开始日期 |
| PRJ-RULE-03 | 项目成员数据隔离 | 只有项目成员可以查看项目 |
| PRJ-RULE-04 | 项目删除约束 | 有任务的项目不能删除 |
| PRJ-RULE-05 | 项目状态管理 | 状态流转规则 |
| PRJ-RULE-06 | 版本冲突检测 | 更新时检测版本号，冲突返回409 |
| PRJ-RULE-07 | 版本历史记录 | 每次更新记录版本历史 |

**数据隔离规则**:
- 用户只能看到自己参与的项目的数据
- 项目之间数据完全隔离
- 系统管理员可以查看所有项目

### 1.4 用户场景

**来源**: FINAL_REQUIREMENTS#L229-284

#### 场景1：创建新项目

**前置条件**:
- 已登录系统
- 具有创建项目权限

**操作步骤**:
1. 点击"新建项目"按钮
2. 填写项目基本信息（项目代号、项目名称、项目类型）
3. 设置项目时间范围（计划开始日期、计划结束日期）
4. 添加项目描述（可选）
5. 点击"创建"按钮

**预期结果**:
- 新项目创建成功
- 显示在项目列表中
- 可以进入项目详情
- 系统记录操作日志

#### 场景2：编辑项目信息

**前置条件**:
- 项目已存在
- 具有编辑权限

**操作步骤**:
1. 进入项目详情
2. 点击"编辑"按钮
3. 修改需要变更的信息
4. 点击"保存"按钮

**预期结果**:
- 项目信息更新成功
- 项目成员收到通知
- 系统记录变更历史
- 版本号自动+1

#### 场景3：管理项目成员

**前置条件**:
- 项目已存在
- 是项目经理或系统管理员

**操作步骤**:
1. 进入项目详情
2. 切换到"成员"标签
3. 点击"添加成员"按钮
4. 从组织架构选择要添加的成员
5. 点击"确认"按钮

**预期结果**:
- 成员添加到项目
- 成员可以访问项目数据
- 成员收到项目邀请通知

**来源**: UI_Requirement#L2507-2596

**里程碑列表表格**:

| 列名 | 宽度 | 说明 |
|------|------|------|
| 状态图标 | w-12 | 已达成(绿)/进行中(蓝)/待处理(灰)/已逾期(红) |
| 名称 | flex | 里程碑名称，可点击编辑 |
| 目标日期 | w-32 | YYYY-MM-DD格式 |

| 状态 | 图标 | 颜色 | 条件 |
|------|------|------|------|
| 待处理 | - | 灰色 | completion_percentage = 0 |
| 进行中 | - | 蓝色 | 0 < completion_percentage < 100 |
| 已达成 | - | 绿色 | completion_percentage = 100 |
| 已逾期 | - | 红色 | 目标日期 < 今天 且 completion_percentage < 100 |

- 里程碑名称：必填，最大100字符

- 描述：可选，多行文本，最大200字符
- 完成百分比：滑块，0-100，默认0

## 3. 时间线管理

### 3.1 功能需求

**来源**: FINAL_REQUIREMENTS#L807-911

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建时间轴 | 为项目创建多条独立时间轴 | P0 |
| 删除时间轴 | 删除指定时间轴及其任务 | P0 |
| 创建任务节点 | 在时间轴上创建任务 | P0 |
| 编辑任务节点 | 修改任务信息 | P0 |
| 删除任务节点 | 删除任务节点 | P0 |
| 拖拽调整时间 | 通过拖拽调整任务时间 | P0 |
| 自动排列 | 智能调整任务位置避免重叠 | P1 |

### 3.2 界面布局规格

**来源**: FINAL_REQUIREMENTS#L817-824

| 区域 | 宽度 | 说明 |
|------|------|------|
| 全局侧边栏 | 256px | 项目导航、用户菜单等 |
| 时间线标签区 | 180px | 显示时间轴名称和任务标签（可拖动调整100-300px） |
| 时间线主区域 | 自适应 | 剩余空间，显示时间刻度和任务条 |

### 3.3 代码增强功能

**来源**: FINAL_REQUIREMENTS#L825-834

| 增强功能 | 描述 |
|---------|------|
| 可调整大小对话框 | 拖拽边缘调整大小，拖拽标题栏移动 |
| 尺寸记忆 | 记忆用户调整的对话框尺寸 |
| 自动排列 | 智能计算任务间隔，避免重叠 |
| 缩放功能 | 支持日/周/月三种视图级别 |
| ESC取消机制 | 任何拖拽操作可按ESC取消 |
| 右键菜单 | 编辑/复制/切换状态/删除 |

### 3.4 业务规则

**来源**: FINAL_REQUIREMENTS#L888-911

#### 规则1：时间轴独立性
- 每个项目可以创建多条时间轴
- 时间轴之间没有依赖关系
- 时间轴可以并行运行

#### 规则2：任务节点时间约束
- 任务节点不能超出时间轴的起止范围
- 拖拽时自动限制在时间轴范围内
- 如果尝试拖出范围，自动吸附到边界

### 3.5 用户场景

**来源**: FINAL_REQUIREMENTS#L836-887

#### 场景1：创建时间轴

**前置条件**:
- 已登录
- 已创建项目

**操作步骤**:
1. 进入项目详情页
2. 点击"时间线"标签
3. 点击"新建时间轴"按钮
4. 输入时间轴名称、起止日期
5. 点击"保存"

**预期结果**:
- 新时间轴添加到列表
- 在时间线视图中显示为独立轨道

#### 场景2：在时间轴上创建任务节点

**前置条件**:
- 时间轴已存在

**操作步骤**:
1. 切换到时间线视图
2. 在时间轴上双击空白区域
3. 弹出任务节点编辑对话框
4. 填写任务信息（名称、开始时间、结束时间）
5. 点击"保存"

**预期结果**:
- 任务节点显示在时间轴上
- 可以拖拽调整时间
- 关联到WBS任务（如果选择了）

#### 场景3：拖拽调整任务时间

**前置条件**:
- 任务节点已存在

**操作步骤**:
1. 鼠标悬停在任务节点上
2. 按住左键开始拖拽
3. 移动到新的时间位置
4. 松开鼠标确认

**预期结果**:
- 任务时间自动更新
- 相邻任务自动调整位置（如果开启了自动排列）
- 关联的WBS任务时间同步更新
