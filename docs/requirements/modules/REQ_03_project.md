## 1. 项目管理

### 1.1 项目CRUD（保持不变）

| 功能 | 描述 | 优先级 | API端点 |
|------|------|--------|---------|
| 创建项目 | 填写项目信息并创建 | P0 | POST /api/projects |
| 编辑项目 | 修改项目信息（带版本控制） | P0 | PUT /api/projects/v2/:id |
| 删除项目 | 删除项目（需确认） | P0 | DELETE /api/projects/v2/:id |
| 查看项目列表 | 查看所有有权限的项目 | P0 | GET /api/projects |
| 查看项目详情 | 查看项目完整信息 | P0 | GET /api/projects/:id/detail |

### 1.2 项目成员管理（更新）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 添加项目成员 | 从组织架构选择成员添加到项目 | P0 |
| 移除项目成员 | 从项目中移除成员 | P0 |
| 查看项目成员 | 查看项目成员列表 | P0 |

**API端点**: `GET/POST/DELETE /api/projects/:id/members`

#### 权限控制（更新）

| 操作 | 所需系统角色 |
|------|-------------|
| 添加项目成员 | admin 或 dept_manager 或 tech_manager |
| 移除项目成员 | admin 或 dept_manager 或 tech_manager |
| 查看项目成员 | 项目成员 |

**验收标准（更新）**:
- admin/dept_manager/tech_manager 可以添加/移除成员
- 成员添加后可立即访问项目数据
- 移除成员后该成员无法访问项目数据

### 1.3 业务规则（更新）

| 规则ID | 规则名称 | 描述 |
|--------|---------|------|
| PRJ-RULE-01 | 项目类型关联 | 项目类型关联任务类型 |
| PRJ-RULE-02 | 项目时间约束 | 结束日期不能早于开始日期 |
| PRJ-RULE-03 | 项目成员数据隔离 | 只有项目成员可以查看项目 |
| PRJ-RULE-04 | 项目删除约束 | 有任务的项目不能删除 |
| PRJ-RULE-05 | 项目状态管理 | 状态流转规则 |
| PRJ-RULE-06 | 版本冲突检测 | 更新时检测版本号，冲突返回409 |
| PRJ-RULE-07 | 版本历史记录 | 每次更新记录版本历史 |

**数据隔离规则（更新）**:
- 用户只能看到自己参与的项目的数据
- 项目之间数据完全隔离
- 系统管理员、部门经理可以查看本范围内所有项目
- 技术经理可以查看本技术组涉及的所有项目

## 2. 项目成员模型（更新）

### 2.1 项目成员表结构

| 字段名称 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| project_id | UUID | 是 | 关联项目 |
| user_id | Integer | 是 | 关联用户 |
| joined_at | Timestamp | 是 | 加入时间 |

**说明**:
- ~~移除 role 字段~~ - 项目成员不再区分角色
- 权限控制由系统角色（users.role）决定
- 同一用户在同一项目中只能有一条记录

### 2.2 项目成员与系统角色的关系

| 系统角色 | 项目成员管理权限 |
|----------|-----------------|
| admin | 可以管理所有项目的成员 |
| dept_manager | 可以管理本部门涉及项目的成员 |
| tech_manager | 可以管理本技术组涉及项目的成员 |
| engineer | 无管理权限，只能查看 |
