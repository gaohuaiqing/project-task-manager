## 4. 数据访问规则（保持不变）

### 规则1：项目数据隔离

- 只能看到自己参与的项目的数据
- 项目成员关系由系统角色控制（admin/dept_manager/tech_manager）
- 系统管理员可以查看所有数据

### 规则2：审批权限

- 工程师修改计划 → 需要审批
- 技术经理修改 → 直接生效
- 部门经理修改 → 直接生效
- 系统管理员修改 → 直接生效

### 规则3：数据访问范围明细

| 角色 | 数据访问范围 | 人员管理范围 |
|------|-------------|-------------|
| admin | 所有项目、任务 | 所有人员 |
| dept_manager | 本部门所有人员涉及的项目、任务 | 本部门所有人员 |
| tech_manager | 本技术组所有人员涉及的项目、任务 | 本技术组人员 |
| tech_manager（授权） | 部门经理授权的其他技术组的项目、任务 | 授权范围内的其他技术组人员 |
| engineer | 参与的项目、任务 | 无 |

## 6. 权限说明

### 6.1 tech_manager 权限边界

技术经理的职责是**管理技术组内的项目和任务**，不包括：
- 组织架构管理（部门、成员的增删）
- 系统设置管理（能力模型、任务类型、节假日配置）
- 审计日志查看
- 技术组授权管理

### 6.2 dept_manager 权限边界

部门经理的职责是**管理部门整体的人、事、设置**，包括：
- 组织架构管理（部门、成员的增删改查）
- 项目和任务管理
- 系统设置管理（能力模型、任务类型、节假日配置）
- 审计日志查看
- 技术组授权管理

### 6.3 工程师任务权限详细说明

#### 6.3.1 任务操作权限总览

| 操作 | 权限 | 审批要求 |
|------|:----:|----------|
| **创建根任务** | ❌ 不允许 | - |
| **创建子任务** | ✅ 允许 | 不需要审批 |
| **编辑非计划字段** | ✅ 允许 | 不需要审批 |
| **编辑计划字段** | ✅ 允许 | 需要审批 |
| **删除任务** | ❌ 不允许 | - |
| **分配任务** | ❌ 不允许 | - |

#### 6.3.2 创建子任务规则

| 规则 | 说明 |
|------|------|
| 父任务限制 | 只能在**已分配给自己**的任务下创建子任务 |
| 继承属性 | 子任务继承父任务的项目、WBS路径 |
| 直接生效 | 创建后直接生效，无需审批 |
| API验证 | 创建时检查 `parent_task.assignee_id === current_user.id` |

#### 6.3.3 编辑任务审批规则

| 项目 | 说明 |
|------|------|
| **需审批字段** | 开始日期、结束日期、工期、前置任务、提前/落后天数 |
| **审批人** | 直接主管（从组织架构读取，无则默认技术经理） |
| **审批人兜底规则** | 无主管 → 技术经理 → 部门经理 → 系统管理员（逐级向上） |
| **审批流程** | 提交 → 待审批状态 → 主管审批 → 通过/驳回 |
| **其他角色** | tech_manager/dept_manager/admin 修改直接生效 |
| **任务删除** | 管理者删除任务时不通知子任务创建者，但系统记录审计日志（操作时间、 操作人、 目标任务、 删除原因) |

个人资料页面各字段的编辑权限：

| 字段 | 用户自己 | admin | dept_manager | tech_manager | engineer |
|------|:--------:|:-----:|:------------:|:-----------:|:--------:|
| 用户名 | 👁️ 只读 | ✅ 编辑 | ✅ 编辑* | 👁️ 只读 | 👁️ 只读 |
| 显示名称 | 👁️ 只读 | ✅ 编辑 | ✅ 编辑* | 👁️ 只读 | 👁️ 只读 |
| 邮箱 | 👁️ 只读 | ✅ 编辑 | ✅ 编辑* | 👁️ 只读 | 👁️ 只读 |

- `*` dept_manager 只能编辑本部门成员的资料

## 8. 用户管理

### 8.1 用户生命周期

**创建用户：**

| 功能 | 描述 |
|------|------|
| 权限要求 | 管理员 或 有权限创建组织架构的角色 |
| 创建方式 | 在组织架构中添加成员时自动创建账户 |
| 必填字段 | 姓名、工号、角色 |
| 默认密码 | 自动生成，方便拷贝（一次性显示） |

**用户创建流程：**
```
1. 在组织架构中选择部门
2. 点击"添加成员"
3. 填写姓名、工号
4. 系统自动创建账户，生成默认密码
5. 弹窗显示账户信息（用户名=工号，默认密码）
6. 可一键拷贝账户信息
```

### 8.2 个人信息管理

| 功能 | 描述 |
|------|------|
| 查看个人信息 | 当前登录用户的资料 |
| 修改个人信息 | 姓名 |
| 修改密码 | 用户自助修改（需验证旧密码） |

### 8.3 密码管理

| 功能 | 描述 |
|------|------|
| 管理员重置密码 | 管理员可重置任意用户密码 |
| 初始密码生成 | 创建用户时自动生成，方便拷贝 |
| 密码复制功能 | 提供一键复制按钮 |

### 8.4 工号管理

| 功能 | 描述 |
|------|------|
| 工号格式 | 8位数字 或 6位字母+数字 |
| 唯一性验证 | 工号不可重复 |
| 作为登录账号 | 工号即用户名 |

### 8.5 用户与组织架构关联规则

| 用户类型 | 创建方式 | 组织架构要求 |
|----------|----------|--------------|
| **内置用户** | 系统初始化时创建 | 无需关联组织架构 |
| **普通用户** | 通过组织架构创建 | 必须先在组织架构中存在 |

**内置用户定义：**

| 用户名 | 角色 | 用途 |
|--------|------|------|
| admin | admin | 系统管理员，用于系统初始化和紧急维护 |

**内置用户特征：**
- 由数据库迁移脚本在系统初始化时创建
- 不需要关联部门（`department_id` 可为 null）
- 不可通过常规成员管理流程删除
- 首次登录后应强制修改密码
