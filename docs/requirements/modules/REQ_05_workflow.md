## 1. 计划变更管理

### 1.1 功能概述

**来源**: FINAL_REQUIREMENTS#L1373-1382

管理任务计划的变更记录和延期状态，包括变更历史追溯、延期原因记录、自动延期检测等功能。

### 1.2 变更记录规则

**来源**: FINAL_REQUIREMENTS#L1436-1470

#### 哪些字段变更会记录
- 开始日期
- 结束日期
- 工期
- 前置任务
- 提前/落后天数

#### 哪些字段变更不会记录
- 任务描述
- 任务负责人
- 任务优先级

### 1.3 审批规则

**来源**: FINAL_REQUIREMENTS#L1449-1463

| 角色 | 修改后 |
|------|--------|
| 工程师 | 需要审批 |
| 技术经理 | 直接生效 |
| 系统管理员 | 直接生效 |

### 1.4 变更次数统计

**来源**: FINAL_REQUIREMENTS#L1457-1463

- 每次提交变更，计数+1
- 包括待审批、已通过、已驳回的变更
- 驳回后重新提交，计数继续增加

### 1.5 变更原因必填

**来源**: FINAL_REQUIREMENTS#L1464-1470

- 工程师提交变更时必须填写变更原因
- 变更原因不能为空
- 变更原因不能超过500字

### 1.6 用户场景

#### 场景1：工程师提交计划变更

**来源**: FINAL_REQUIREMENTS#L1385-1403

**前置条件**:
- 您是工程师角色
- 任务已存在

**操作步骤**:
1. 编辑任务计划（修改日期、工期等）
2. 点击"保存"按钮
3. 填写变更原因（必填）
4. 点击"提交审批"按钮
5. 等待审批

**预期结果**:
- 变更请求已成功提交
- 任务状态变为"待审批"
- "计划调整"计数+1
- 审批人收到通知

#### 场景2：技术经理直接修改计划

**来源**: FINAL_REQUIREMENTS#L1405-1418

**前置条件**:
- 您是技术经理或系统管理员角色
- 任务已存在

**操作步骤**:
1. 编辑任务计划
2. 点击"保存"按钮

**预期结果**:
- 修改立即生效
- 变更自动记录到历史
- "计划调整"计数+1

#### 场景3：查看变更历史

**来源**: FINAL_REQUIREMENTS#L1420-1433

**前置条件**:
- 任务有变更历史

**操作步骤**:
1. 找到"计划调整"列
2. 点击"计划调整"列中的数字
3. 查看历史记录

**预期结果**:
- 显示完整的变更历史
- 可以看到每次变更的详细信息
- 包含：变更时间、变更人、变更内容、变更原因

## 2. 审批流程

### 2.1 功能概述

**来源**: FINAL_REQUIREMENTS#L1071-1080

管理计划变更的审批流程，包括提交审批、审核变更、查看历史等功能。确保计划变更得到适当授权。

### 2.2 审批流程增强

**来源**: FINAL_REQUIREMENTS#L1081-1088

| 增强功能 | 描述 | 来源 |
|---------|------|------|
| 审批超时处理 | 超过7天自动标记，红色高亮 | 代码 |
| 批量审批 | 支持选择多个任务批量审批 | 代码 |
| 统计功能 | 统计超时任务数 | 代码 |

### 2.3 用户场景

#### 场景1：工程师提交计划变更

**来源**: FINAL_REQUIREMENTS#L1091-1110

**前置条件**:
- 您是工程师角色
- 有可编辑的任务

**操作步骤**:
1. 编辑任务计划（修改开始日期、结束日期、工期等）
2. 点击"保存"按钮
3. 系统弹出变更原因输入框
4. 填写变更原因
5. 点击"提交审批"按钮
6. 等待审批

**预期结果**:
- 变更请求已提交
- 任务状态变为"待审批"
- 审批人收到通知
- 您可以在审批列表中查看进度

#### 场景2：主管审核变更

**来源**: FINAL_REQUIREMENTS#L1112-1128

**前置条件**:
- 您是审批人（技术经理或直属主管）
- 有待审批的变更

**操作步骤**:
1. 打开审批列表
2. 选择"待审批"标签
3. 点击某个变更项
4. 查看变更详情（变更前后对比）
5. 做出审批决策（通过/驳回）
6. 如果驳回，填写驳回原因

**预期结果**:
- **通过**：任务计划更新，记录变更次数，申请人收到通知
- **驳回**：变更不生效，申请人收到驳回通知和原因

#### 场景3：查看审批历史

**来源**: FINAL_REQUIREMENTS#L1130-1143

**前置条件**:
- 您有审批记录

**操作步骤**:
1. 打开审批列表
2. 选择"我的审批"或"我发起的"标签
3. 查看所有审批记录

**预期结果**:
- 可以查看完整的审批历史
- 可以查看每个审批的详细信息
- 支持按状态、时间筛选

### 2.4 业务规则

#### 规则1：审批触发条件

**来源**: FINAL_REQUIREMENTS#L1146-1158

**规则说明**:
- 只有工程师修改计划时才需要审批
- 技术经理修改直接生效
- 系统管理员修改直接生效
- 需要审批的字段：开始日期、结束日期、工期、前置任务、提前/落后天数

#### 规则2：审批人确定

**来源**: FINAL_REQUIREMENTS#L1159-1168

**规则说明**:
- 审批人是直接主管（从组织架构读取）
- 如果没有设置主管，默认为技术经理
- 审批人收到实时通知

#### 规则3：审批有效期

**来源**: FINAL_REQUIREMENTS#L1169-1179

**规则说明**:
- 审批请求有效期7天
- 超过有效期未审批，自动标记超时
- 超时任务红色高亮显示

#### 规则4：审批通过生效

**来源**: FINAL_REQUIREMENTS#L1180-1191

**规则说明**:
- 审批通过后，变更自动生效
- 任务计划更新为变更后的值
- "计划调整"计数+1
- 通知申请人

#### 规则5：审批驳回处理

**来源**: FINAL_REQUIREMENTS#L1192-1203

**规则说明**:
- 审批驳回后，变更不生效
- 任务保持原值
- 工程师收到驳回通知和原因
- 工程师可以修改后重新提交

## 3. 延期管理

### 3.1 延期自动判断与智能累计

**来源**: FINAL_REQUIREMENTS#L1522-1562

#### 判断规则
- 系统每日凌晨1点自动检查所有任务
- 判断条件：当前日期 > 计划结束日期 且 任务状态不是"已完成"
- 自动标记为"已延迟"

#### 延期次数智能累计规则
- **首次延期**：任务首次超过计划结束日期时，延期次数 = 1
- **持续延期不累加**：计划未刷新期间，延期次数保持不变
- **刷新后再次超期**：用户刷新计划后，任务再次超过新的结束日期时，延期次数 +1
- **计数含义**：延期次数记录的是"延期了几轮"，而非"延期了多少天"

### 3.2 延期原因记录

**来源**: FINAL_REQUIREMENTS#L1564-1577

**规则说明**:
- 工程师和技术经理可以添加延期原因
- 延期原因只能增加，不能删除或修改
- 每条记录包含：延期时间、延期原因、延期天数、记录人

### 3.3 延期天数计算

**来源**: FINAL_REQUIREMENTS#L1578-1591

**规则说明**:
- 延期天数 = 当前日期 - 计划结束日期（天数）
- 不足1天按1天计算
- 每日自动更新延期天数

### 3.4 延期状态显示

**来源**: FINAL_REQUIREMENTS#L1593-1605

| 状态 | 颜色 | 图标 |
|------|------|------|
| 已延迟 | 红色 | ⚠️ |
| 延期预警 | 橙色 | ⏰ |
| 正常 | 默认 | - |

### 3.5 延期通知规则

**来源**: FINAL_REQUIREMENTS#L1606-1621

**规则说明**:
- 任务首次延期时发送通知
- 延期天数每增加7天再次通知
- 通知发送给：任务负责人、项目经理、技术经理

## 接口需求

### API列表

#### 计划变更
```
GET    /api/tasks/:id/changes         # 获取任务变更历史
POST   /api/tasks/:id/changes         # 提交计划变更
```

#### 审批流程
```
GET    /api/approvals                 # 获取审批列表
GET    /api/approvals/pending         # 获取待审批列表
POST   /api/approvals/:id/approve     # 通过审批
POST   /api/approvals/:id/reject      # 驳回审批
```

#### 延期记录
```
GET    /api/tasks/:id/delays          # 获取任务延期记录
POST   /api/tasks/:id/delays          # 添加延期原因
```
