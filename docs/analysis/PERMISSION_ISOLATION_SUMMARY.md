## 一、角色定义与数据访问范围

### 1.1 角色定义

| 角色 | 中文名称 | 描述 | 可编辑WBS | 数据访问范围 |
|------|----------|------|:---------:|--------------|
| **admin** | 系统管理员 | 系统最高权限 | ✅ | 所有项目数据 |
| **dept_manager** | 部门经理 | 业务部门管理者 | ✅ | 本部门项目数据 |
| **tech_manager** | 技术经理 | 技术部门管理者 | ✅ | 本技术组项目数据 |
| **engineer** | 工程师 | 普通开发人员 | ✅* | 参与的项目数据 |

*注：工程师编辑计划相关字段（开始日期、结束日期、工期、前置任务、提前/落后天数）需要审批

### 1.2 组织架构关系

```
部门
├── 部门经理（dept_manager）
│   ├── 技术组1
│   │   ├── 技术经理（tech_manager）
│   │   ├── 工程师A
│   │   └── 工程师B
│   ├── 技术组2
│   │   ├── 技术经理（tech_manager）
│   │   ├── 工程师C
│   │   └── 工程师D
│   └── ...
```

### 1.3 数据访问范围明细

| 角色 | 数据访问范围 | 人员管理范围 | 报表分析数据范围 |
|------|-------------|-------------|------------------|
| admin | 所有项目、任务 | 所有人员 | 所有人员的任务数据 |
| dept_manager | 本部门所有人员涉及的项目、任务 | 本部门所有人员 | 本部门人员的任务数据 |
| tech_manager | 本技术组所有人员涉及的项目、任务 | 本技术组人员 | 本技术组 + 授权技术组人员的任务数据 |
| tech_manager（授权） | 部门经理授权的其他技术组的项目、任务 | 授权范围内的其他技术组人员 | 授权范围内的任务数据 |
| engineer | 参与的项目、任务 | 无 | 不可访问报表分析 |

## 二、仪表板权限隔离

### 2.1 模块可见性

| 角色 | 可见仪表板模块 |
|------|:-------------:|
| admin | ✅ |
| dept_manager | ✅ |
| tech_manager | ✅ |
| engineer | ✅ |

### 2.2 统计卡片数据范围（角色差异化）

| 卡片 | admin/dept_manager/tech_manager | engineer |
|------|--------------------------------|----------|
| 卡片1 | 项目总数（关注范围内） | 参与项目数 |
| 卡片2 | 进行中任务数 | 我的进行中任务数 |
| 卡片3 | 已完成任务数 | 我的已完成任务数 |
| 卡片4 | 延期预警数 | 我的到期/逾期任务数 |

### 2.3 组件显示差异

**管理者角色（admin/dept_manager/tech_manager）**：
- 显示全部7个组件
- 数据范围按角色过滤
- 包含：任务分布组件、状态饼图组件等分析类组件

**工程师角色（engineer）**：
- 显示统计卡片（个人数据）
- 紧急任务提醒
- 需要更新的任务（超过7天未更新进展的任务）
- 项目进度（仅自己参与的项目）
- **不显示**：任务列表（WBS表已有）、分布/饼图类分析组件

### 2.4 角色差异化仪表板内容

#### admin（系统管理员）仪表板

**定位**：全局汇总 + 部门对比 + 资源调配建议

| 组件 | 类型 | 说明 |
|------|------|------|
| 风险预警区 | 预警卡片 | 延期预警 + 已延期 + 待审批 + 高风险项目 |
| 核心指标卡片 | 8个统计卡片 | 项目数、任务数、完成率、延期率、人数、利用率、里程碑、本周到期 |
| 部门效能对比 | 可排序表格 | 横向对比各部门的完成率、延期率、利用率、状态 |
| 部门延期率趋势 | 多线图 | 各部门延期率的时间变化趋势 |
| 资源调配建议 | 建议卡片 | 智能识别资源瓶颈和调配建议 |

#### dept_manager（部门经理）仪表板

**定位**：部门汇总 + 组/人员对比 + 资源调配建议

| 组件 | 类型 | 说明 |
|------|------|------|
| 部门风险预警区 | 预警卡片 | 本部门的延期预警 + 已延期 + 待审批 |
| 部门核心指标卡片 | 8个统计卡片 | 部门项目数、任务数、完成率、延期率、人数、利用率、本周到期、活跃度 |
| 组效能对比表格 | 可排序表格 | 横向对比本部门各组的完成率、延期率、负载率、活跃度 |
| 任务类型分布 | 柱状图 | 12种任务类型的数量分布 |
| 成员负载分布 | 散点图 | X轴任务数、Y轴完成率，识别异常成员 |
| 人员调配建议 | 建议卡片 | 智能识别过载成员、闲置成员、低活跃成员 |

#### tech_manager（技术经理）仪表板

**定位**：组汇总 + 成员对比 + 任务分配建议

| 组件 | 类型 | 说明 |
|------|------|------|
| 组切换下拉框 | 导航控件 | 被授权管理多个技术组时显示，可切换查看不同组 |
| 置顶预警卡片 | 4个预警卡片 | 延期预警 + 已延期 + 待我审批 + 待分配 |
| 组内核心指标卡片 | 8个统计卡片 | 组内项目数、任务数、完成率、延期率、人数、平均负载、本周到期、活跃度 |
| 成员任务状态表格 | 可排序表格 | 横向对比组内成员的任务数量、完成率、负载率、活跃度 |
| 成员任务分布图 | 堆叠柱状图 | 展示每个成员的任务状态分布 |
| 任务分配建议 | 建议卡片 | 智能识别过载成员、可接单成员、低活跃成员 |

#### engineer（工程师）仪表板

**定位**：纯个人数据 - 只看自己的任务和项目

| 组件 | 类型 | 说明 |
|------|------|------|
| 我的紧急任务区 | 3个预警卡片 | 逾期 + 今日到期 + 本周到期 |
| 我的待办任务列表 | 任务列表 | 紧随置顶区，按优先级排序 |
| 需要更新的任务 | 提醒卡片 | 显示超过7天未更新进展的任务 |
| 我的核心指标卡片 | 4个统计卡片 | 参与项目数、进行中、已完成、待开始 |
| 我的任务趋势图 | 折线图 | 30天个人任务完成/新增趋势 |
| 我的任务状态分布 | 饼图 | 未开始/进行中/已完成 |
| 参与项目进度 | 列表 | 只显示自己参与的项目及其进度 |

## 四、WBS任务表可见任务规则

### 4.1 项目数据隔离规则

> **核心规则**：只能看到自己参与的项目的数据

| 角色 | 可见任务范围 |
|------|-------------|
| admin | 所有项目的所有任务 |
| dept_manager | 本部门人员参与的项目的所有任务 |
| tech_manager | 本技术组人员参与的项目 + 被授权技术组的项目任务 |
| engineer | 自己参与的项目任务 |

### 4.2 任务操作权限矩阵

| 操作 | admin | dept_manager | tech_manager | engineer |
|------|:-----:|:------------:|:-----------:|:--------:|
| 创建根任务 | ✅ | ✅ | ✅ | ❌ |
| 创建子任务 | ✅ | ✅ | ✅ | ✅* |
| 编辑非计划字段 | ✅ | ✅ | ✅ | ✅ |
| 编辑计划字段 | ✅ | ✅ | ✅ | ✅** |
| 删除任务 | ✅ | ✅ | ✅ | ❌ |
| 分配任务 | ✅ | ✅ | ✅ | ❌ |
| 批量删除任务 | ✅ | ✅ | ❌ | ❌ |

*工程师只能在自己负责的任务下创建子任务
**工程师编辑计划字段需要审批

### 4.3 工程师任务权限详细规则

#### 创建子任务规则

| 规则 | 说明 |
|------|------|
| 父任务限制 | 只能在**已分配给自己**的任务下创建子任务 |
| 继承属性 | 子任务继承父任务的项目、WBS路径 |
| 直接生效 | 创建后直接生效，无需审批 |
| API验证 | 创建时检查 `parent_task.assignee_id === current_user.id` |

#### 编辑任务审批规则

| 项目 | 说明 |
|------|------|
| **需审批字段** | 开始日期、结束日期、工期、前置任务、提前/落后天数 |
| **审批人** | 直接主管（从组织架构读取，无则默认技术经理） |
| **审批人兜底规则** | 无主管 → 技术经理 → 部门经理 → 系统管理员（逐级向上） |
| **审批流程** | 提交 → 待审批状态 → 主管审批 → 通过/驳回 |
| **其他角色** | tech_manager/dept_manager/admin 修改直接生效 |

#### 字段分类

| 类别 | 包含字段 | 审批要求 |
|------|----------|----------|
| **非计划字段** | 任务描述、负责人、优先级、预警天数、实际开始、实际结束、全职比、Redmine链接 | 不需要审批 |
| **计划字段** | 开始日期、结束日期、工期、前置任务、提前/落后天数 | **需要审批** |

#### 权限跟随负责人

| 规则 | 说明 |
|------|------|
| 场景 | 任务负责人从工程师A变更为工程师B |
| 规则 | 工程师A失去对子任务Y的编辑权限，新负责人工程师B获得权限 |

## 七、实现要点

### 7.1 前端实现

- 侧边栏根据用户角色过滤菜单项
- 仪表板组件根据角色差异化显示
- 报表分析模块对 engineer 隐藏

### 7.2 后端实现

- API 已有权限验证，根据用户角色返回对应范围的数据
- 数据隔离基于项目成员关系 + 组织架构层级
- 审批流程根据角色判断是否需要审批

### 7.3 数据隔离逻辑

```
1. 项目级隔离：用户只能看到自己参与的项目
2. 组织架构级隔离：
   - admin: 所有数据
   - dept_manager: 本部门数据
   - tech_manager: 本技术组 + 授权技术组数据
   - engineer: 个人数据
3. 组合过滤：项目成员关系 ∩ 组织架构范围
```

## 八、关键发现

需求文档中对权限隔离的定义是**完整且清晰**的：

1. **仪表板**：按角色差异化显示组件和数据范围
2. **报表分析**：engineer不可见，其他角色按组织架构范围过滤数据
3. **WBS任务**：基于项目成员关系过滤，结合组织架构层级
4. **细粒度权限**：17种权限项，覆盖组织架构、项目、成员、任务、系统、报表分析六大类别
