# AI 自动化测试方案

> **版本**: 3.0
> **日期**: 2026-04-06
> **目标**: 定义AI执行的自动化测试范围、策略、数据和预期结果

## 一、测试目标与范围

### 1.1 测试目标

| 目标 | 说明 |
|------|------|
| **功能验证** | 验证所有功能模块符合需求规格 |
| **权限验证** | 验证4种角色的权限边界正确 |
| **性能验证** | 验证系统满足性能目标（响应<300ms，100人并发） |
| **数据一致性** | 验证并发操作下数据不丢失、不冲突 |
| **回归保障** | 建立可重复执行的测试基线 |

### 1.2 测试范围

| 模块 | 功能测试 | 性能测试 | 并发测试 | 优先级 | 用例数 |
|------|----------|----------|----------|--------|--------|
| 登录认证 | ✅ | ✅ | ✅ | P0 | 23 |
| 项目管理 | ✅ | ✅ | ✅ | P0 | 48 |
| 任务管理 | ✅ | ✅ | ✅ | P0 | 58 |
| 审批流程 | ✅ | - | ✅ | P0 | 26 |
| 组织架构 | ✅ | - | - | P1 | 32 |
| 仪表板 | ✅ | ✅ | - | P0 | 18 |
| 报表分析 | ✅ | ✅ | - | P1 | 37 |
| 系统设置 | ✅ | - | - | P2 | 36 |
| **总计** | - | - | - | - | **370** |

### 1.3 不在范围内

- 安全渗透测试
- 跨浏览器兼容性测试
- 移动端适配测试
- 数据库压力测试

## 二、测试策略

### 2.1 三层测试架构

```
┌─────────────────────────────────────────────┐
│           L3 全业务流测试                     │
│  (端到端、真实用户视角、完整业务场景)          │
├─────────────────────────────────────────────┤
│           L2 小模块集成测试                   │
│  (验证模块间协作正确性)                       │
├─────────────────────────────────────────────┤
│           L1 模块用例测试                     │
│  (验证单个功能点正确性)                       │
└─────────────────────────────────────────────┘
```

| 层级 | 目的 | 用例特点 | 文档位置 |
|------|------|----------|----------|
| **L1 模块用例** | 验证单个功能点正确性 | 独立、原子性、可并行 | `test-cases/module/` |
| **L2 集成测试** | 验证模块间协作正确性 | 有依赖、需顺序执行 | `test-cases/integration/` |
| **L3 业务流测试** | 验证完整业务场景 | 端到端、真实用户视角 | `test-cases/e2e/` |

### 2.2 测试执行方式

**技术选型**: Chrome DevTools MCP

**选择理由**:
1. 项目已集成 Chrome DevTools MCP
2. AI 可直接操作浏览器，无需额外框架
3. 支持多 Browser Context 实现并发测试
4. 支持性能追踪和 Lighthouse 审计

### 2.3 测试数据策略

| 策略 | 说明 |
|------|------|
| **数据前缀** | 所有测试数据以 `E2E_` 开头 |
| **独立数据库** | 使用 `task_manager_test` 测试数据库 |
| **数据隔离** | 每次测试前清理旧数据，测试后清理新数据 |
| **真实用户** | 使用系统中已存在的测试用户 |

## 三、测试用户与权限

### 3.1 测试用户获取策略

> **原则**: 不预设测试账户，使用数据库中已存在的用户进行测试

**获取方式**：
1. 测试开始前，通过 API 或数据库查询获取已有用户列表
2. 按角色类型筛选所需用户
3. 用户密码统一从系统配置或数据库获取

**所需角色类型**：

| 角色编码 | 角色名称 | 用途 |
|----------|----------|------|
| admin | 系统管理员 | 全权限测试 |
| tech_manager | 技术经理 | 技术经理权限测试 |
| dept_manager | 部门经理 | 部门经理权限测试、跨部门数据隔离测试 |
| engineer | 工程师 | 工程师权限测试、跨部门数据隔离测试 |

**用户查询示例**：
```sql
-- 查询各角色用户
SELECT id, username, name, role, department_id
FROM users
WHERE is_active = 1
ORDER BY role, department_id;
```
