## TC-COLL-04：两人同时编辑同一任务（版本冲突）

### 场景描述

两个用户同时编辑同一任务，验证版本冲突处理

### 前置条件

- 准备两个浏览器窗口
- admin 和 tech_manager 同时登录
- 存在同一个任务可供编辑

### 操作步骤与验证点

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 浏览器 A：admin 打开任务 X 的编辑模式 | 进入编辑模式，记录当前版本号 |
| 2 | 浏览器 B：tech_manager 打开任务 X 的编辑模式 | 进入编辑模式 |
| 3 | 浏览器 A：admin 修改任务描述并保存 | 保存成功，版本号 +1 |
| 4 | 浏览器 B：tech_manager 修改任务描述并保存 | 提示"数据已被他人修改，请刷新后重新编辑" |
| 5 | 浏览器 B：tech_manager 刷新页面 | 显示 admin 的修改内容 |
| 6 | 浏览器 B：tech_manager 重新编辑并保存 | 保存成功 |

### 验证要点

- 系统使用乐观锁机制（版本号）防止数据覆盖
- 后提交的用户必须先刷新获取最新数据
- 数据不会丢失或被错误覆盖

## TC-COLL-05：删除前置任务后的依赖处理

### 场景描述

删除被依赖的前置任务后，验证后续任务的处理

### 前置条件

- 任务 B 依赖任务 A（前置任务关系已建立）

### 操作步骤与验证点

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 删除任务 A | 弹出警告提示"有任务依赖此任务" |
| 2 | 确认删除 | 任务 A 删除成功 |
| 3 | 检查任务 B 的前置任务字段 | 前置任务字段清空 |
| 4 | 检查任务 B 的开始日期 | 恢复为可编辑状态，不再自动计算 |
| 5 | 检查任务 B 的状态 | 状态正常，无异常 |
