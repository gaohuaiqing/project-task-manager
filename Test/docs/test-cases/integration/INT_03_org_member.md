# 集成测试：组织→成员管理

> **场景**: 组织架构与成员管理联动
> **用例数**: 6
> **层级**: L2 集成测试

---

## INT-ORG-MEM-01: 添加成员自动创建账号

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证在组织架构添加成员时自动创建登录账号 |
| **前置条件** | 已登录为管理员 |
| **操作步骤** | 1. 进入组织架构 → 2. 选择部门 → 3. 添加成员 → 4. 保存 |
| **预期结果** | 成员添加成功，显示初始密码 |
| **验证点** | 可用工号和初始密码登录系统 |

---

## INT-ORG-MEM-02: 成员部门变更影响数据权限

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证成员部门变更后数据权限变化 |
| **前置条件** | 成员在研发部，有研发部项目数据 |
| **操作步骤** | 1. 将成员部门改为产品部 → 2. 成员重新登录 → 3. 查看项目列表 |
| **预期结果** | 可见产品部的项目数据，不可见研发部数据 |
| **验证点** | 项目列表只显示产品部项目 |

## INT-ORG-MEM-04: 停用成员后无法登录

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证停用成员后账号立即失效 |
| **前置条件** | 成员处于活跃状态 |
| **操作步骤** | 1. 停用成员 → 2. 成员尝试登录 |

| **验证点** | 显示"账号已停用，请联系管理员" |
