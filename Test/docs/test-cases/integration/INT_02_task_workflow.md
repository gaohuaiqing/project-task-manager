# 集成测试：任务→审批流程

> **场景**: 任务变更触发审批流程
> **用例数**: 7
> **层级**: L2 集成测试

---

## INT-TASK-WF-01: 工程师变更任务触发审批

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证工程师变更计划字段触发审批流程 |
| **前置条件** | 已登录为工程师，有已分配的任务 |
| **操作步骤** | 1. 编辑任务 → 2. 修改工期 → 3. 填写变更原因 → 4. 保存 |
| **预期结果** | 任务状态变为"待审批"，审批列表出现新申请 |
| **验证点** | 审批列表显示变更申请，任务锁定不可再编辑 |

---

## INT-TASK-WF-02: 审批通过后任务自动更新

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证审批通过后任务计划自动更新 |
| **前置条件** | 存在工期变更申请（原10天→新15天） |
| **操作步骤** | 1. 技术经理登录 → 2. 进入审批列表 → 3. 通过申请 |
| **预期结果** | 任务工期更新为15天，状态恢复 |
| **验证点** | 任务详情显示新工期，计划调整数+1 |

---

## INT-TASK-WF-03: 审批驳回后任务恢复原状

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证审批驳回后任务保持原计划 |
| **前置条件** | 存在工期变更申请 |
| **操作步骤** | 1. 技术经理登录 → 2. 驳回申请（填写原因） |
| **预期结果** | 任务工期保持原值，状态恢复 |
| **验证点** | 任务工期仍为原值，申请人收到驳回通知 |

## INT-TASK-WF-04: 技术经理直接修改无需审批

## INT-TASK-WF-05: 依赖任务变更触发级联审批

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证前置任务变更对后续任务的影响 |
| **前置条件** | 任务B依赖任务A，工程师负责任务B |
| **操作步骤** | 1. 技术经理修改任务A的结束日期 → 2. 查看任务B |
| **预期结果** | 任务B的开始日期自动调整 |
| **验证点** | 任务B日期与任务A保持依赖关系 |

## INT-TASK-WF-07: 审批超时后任务状态恢复

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证审批超时（7天）后任务状态恢复 |
| **前置条件** | 存在7天前提交的待审批申请，任务状态为"待审批" |
| **操作步骤** | 1. 等待定时任务执行 → 2. 查看审批状态 → 3. 查看任务状态 |
| **预期结果** | 审批状态变为timeout，任务状态恢复为原状态 |
| **验证点** | 申请人收到超时通知，任务可重新编辑 |
