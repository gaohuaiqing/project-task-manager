# 集成测试：项目→任务流程

> **场景**: 项目创建后任务管理流程
> **用例数**: 7
> **层级**: L2 集成测试

---

## INT-PROJ-TASK-01: 创建项目后添加任务

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证项目创建后可直接添加任务 |
| **前置条件** | 已登录为技术经理 |
| **操作步骤** | 1. 创建项目E2E-INT-001 → 2. 进入项目详情 → 3. 添加根任务 → 4. 添加子任务 |
| **预期结果** | 项目和任务创建成功，任务关联到项目 |
| **验证点** | 任务列表显示在项目下，WBS编码正确 |

## INT-PROJ-TASK-02: 项目成员与任务分配联动

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证只有项目成员可被分配任务 |
| **前置条件** | 存在测试项目 |
| **操作步骤** | 1. 尝试分配非项目成员到任务 → 2. 添加成员到项目 → 3. 再次分配 |

| **验证点** | 成员选择器只显示项目成员 |

## INT-PROJ-TASK-03: 项目删除与任务级联

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证有任务的项目不能删除 |
| **前置条件** | 项目下有任务 |
| **操作步骤** | 1. 尝试删除项目 → 2. 删除所有任务 → 3. 再次删除项目 |
| **预期结果** | 有任务时删除失败，无任务后删除成功 |
| **验证点** | 显示"项目下存在任务，无法删除" |

## INT-PROJ-TASK-04: 项目状态与任务状态联动

| 项目 | 内容 |
|------|------|

| **前置条件** | 项目状态为"规划中"，有任务 |

## INT-PROJ-TASK-06: 项目进度与任务完成率联动

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证项目进度根据任务完成率自动计算 |
| **前置条件** | 项目有3个任务，均未开始 |
| **操作步骤** | 1. 完成1个任务 → 2. 查看项目进度 → 3. 完成2个任务 → 4. 查看项目进度 |
| **预期结果** | 项目进度 = 已完成任务数 / 总任务数 × 100% |
| **验证点** | 完成1个后进度33%，完成2个后进度67% |

## INT-PROJ-TASK-07: 项目归档后任务状态冻结

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证项目归档后任务不可修改 |
| **前置条件** | 项目状态为"进行中"，有未完成任务 |
| **操作步骤** | 1. 将项目状态改为"已归档" → 2. 尝试编辑任务 → 3. 尝试更新进度 |
| **预期结果** | 任务编辑功能禁用，进度更新按钮禁用 |
| **验证点** | 显示"项目已归档，任务不可修改" |
