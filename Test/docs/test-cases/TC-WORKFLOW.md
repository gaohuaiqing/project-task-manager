# 测试用例 - 任务审批流程

> **模块**: Workflow
> **优先级**: P1
> **场景数**: 4

---

## TC-WF-01：工程师提交计划变更审批

### 场景描述

工程师编辑任务计划字段后，提交审批申请

### 前置条件

- 已登录 engineer 用户（50241392）
- engineer 负责的任务存在
- 任务有完整的计划日期

### 操作步骤与验证点

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 进入任务管理页面 | 显示任务列表 |
| 2 | 找到负责人为自己的任务，点击编辑 | 进入编辑模式 |
| 3 | 修改计划字段（开始日期或工期） | 输入新值 |
| 4 | 点击保存 | 提示"已提交审批" |
| 5 | 检查任务状态 | 状态变为"待审批" |
| 6 | 检查审批列表 | 显示待审批记录 |

### 注意事项

- 工程师只能编辑分配给自己的任务

## TC-WF-02：技术经理审批通过

### 场景描述

技术经理审批工程师提交的变更申请

### 前置条件

- 已登录 tech_manager 用户（50234447）
- 存在待审批的变更申请（由 TC-WF-01 创建）

### 操作步骤与验证点

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 进入审批管理页面 | 显示待审批列表 |
| 2 | 点击待审批的变更申请 | 查看变更详情 |
| 3 | 检查变更内容 | 显示变更前后的值对比 |
| 4 | 点击"通过"按钮 | 提示审批成功 |
| 5 | 检查任务状态 | 状态从"待审批"变为"未开始"或"进行中" |
| 6 | 检查任务计划 | 计划已更新为申请的新值 |
| 7 | 检查工程师是否收到通知 | 通知列表显示审批结果 |

## TC-WF-03：技术经理审批驳回

### 场景描述

技术经理驳回工程师提交的变更申请

### 前置条件

- 已登录 tech_manager 用户（50234447）
- 存在待审批的变更申请

### 操作步骤与验证点

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 进入审批管理页面 | 显示待审批列表 |
| 2 | 点击待审批的变更申请 | 查看变更详情 |
| 3 | 点击"驳回"按钮 | 弹出驳回原因输入框 |
| 4 | 输入驳回原因 | 输入成功 |
| 5 | 点击确认驳回 | 提示驳回成功 |
| 6 | 检查任务状态 | 状态从"待审批"变为"已驳回" |
| 7 | 检查任务计划 | 计划保持原值，未变更 |
| 8 | 检查工程师是否收到通知 | 通知列表显示驳回原因 |

## TC-WF-04：审批期间任务锁定

### 场景描述

验证审批期间任务不能被其他操作修改

### 前置条件

- 任务处于"待审批"状态

### 操作步骤与验证点

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 工程师尝试编辑该任务 | 提示"任务审批中，无法编辑" |
| 2 | 另一个工程师尝试编辑该任务 | 提示无权限或任务锁定 |
| 3 | 技术经理尝试编辑该任务 | 可以编辑（审批人有权限） |
