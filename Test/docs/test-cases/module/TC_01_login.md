## 测试用户获取策略

> **原则**: 不预设测试账户，使用数据库中已存在的用户进行测试

**测试前准备**：
1. 查询数据库获取各角色用户：`SELECT id, username, role, department_id FROM users WHERE is_active = 1`
2. 记录用户密码（测试环境统一密码或从配置获取）
3. 确保各角色至少有一个用户存在

**所需角色**：admin, tech_manager, dept_manager, engineer（不同部门）
