## TC-AUTH-01：各角色登录成功

### 场景描述

验证 admin、dept_manager、tech_manager、engineer 四种角色都能正常登录

### 前置条件

- 系统已启动
- 四种角色的测试用户已创建

### 测试数据

| 角色 | 用户名 | 密码 |
|------|--------|------|
| admin | admin | admin123 |
| dept_manager | 50223183 | 50223183 |
| tech_manager | 50234447 | 50234447 |
| engineer | 50241392 | 50241392 |

### 操作步骤与验证点

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 打开登录页面 http://localhost:5173/login | 页面正常加载，显示登录表单 |
| 2 | 输入用户名和密码，点击登录 | 登录成功，跳转到 /dashboard |
| 3 | 检查页面顶部用户信息 | 显示当前登录用户的名称 |
| 4 | 检查左侧菜单 | 根据角色显示对应的菜单项 |

### 角色差异化验证

| 角色 | 应显示菜单 | 不应显示菜单 |
|------|-----------|-------------|
| admin | 全部菜单 | - |
| dept_manager | 仪表板、项目、任务、报表、设置（部分） | 用户管理、系统配置 |
| tech_manager | 仪表板、项目、任务、设置（部分） | 报表、用户管理 |
| engineer | 仪表板、任务、设置（个人） | 报表、项目管理、用户管理 |

## TC-AUTH-02：登录失败锁定

### 场景描述

连续 5 次登录失败后，账户锁定 30 分钟

### 前置条件

- 测试用户 engineer（50241392）已创建且未锁定

### 测试数据

- 用户名：50241392
- 错误密码：wrongpassword

### 操作步骤与验证点

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 使用错误密码登录 50241392 | 提示"用户名或密码错误" |
| 2 | 重复步骤 1 共 5 次 | 每次都提示错误 |
| 3 | 第 6 次尝试登录 | 提示"账户已锁定，请 30 分钟后重试" |
| 4 | 使用正确密码（50241392）登录 | 仍然被拒绝 |
| 5 | 等待 30 分钟后使用正确密码登录 | 登录成功 |

### 注意事项

- 测试完成后需要重置用户锁定状态或等待 30 分钟
