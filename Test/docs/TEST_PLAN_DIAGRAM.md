## 五、E2E 业务流路径图

```
FLOW-01 项目全生命周期（admin）
  创建项目 ──▶ 添加成员 ──▶ 创建里程碑 ──▶ 创建任务 ──▶ 分配任务
      ──▶ 完成任务 ──▶ 验证仪表板 ──▶ 删除项目

FLOW-02 工程师日常工作流（engineer → tech_mgr）
  查看仪表板 ──▶ 查看紧急任务 ──▶ 更新进度 ──▶ 提交变更
      ──▶ [tech_mgr审批] ──▶ 收到通知

FLOW-03 审批完整链路（engineer ↔ tech_mgr）
  提交变更 ──▶ 驳回 ──▶ 重新提交 ──▶ 通过 ──▶ 验证数据

FLOW-04 报表数据一致性（admin）
  记录基线 ──▶ 查看报表 ──▶ 创建数据 ──▶ 验证联动

FLOW-05 新成员入职（admin → engineer）
  创建成员 ──▶ 新人登录 ──▶ 空数据 ──▶ 分配任务 ──▶ 验证

FLOW-06 导入导出（tech_mgr）
  下载模板 ──▶ 填写数据 ──▶ 导入预览 ──▶ 确认导入 ──▶ 导出比对

FLOW-07 系统管理（admin → engineer）
  创建部门 ──▶ 添加用户 ──▶ 配置权限 ──▶ 新用户验证权限
```
