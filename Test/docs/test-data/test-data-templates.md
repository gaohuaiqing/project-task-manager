## 六、测试用户获取

> **原则**: 不预设测试账户，使用数据库中已存在的用户进行测试

### 用户获取方式

1. **API 查询**: `GET /api/users` 或 `GET /api/members`
2. **数据库查询**: `SELECT * FROM users WHERE is_active = 1`

### 所需角色类型

| 角色编码 | 角色名称 | 测试用途 |
|----------|----------|----------|
| admin | 系统管理员 | 全权限测试 |
| tech_manager | 技术经理 | 技术经理权限测试 |
| dept_manager | 部门经理 | 部门经理权限测试 |
| engineer | 工程师 | 工程师权限测试、跨部门测试 |

### 跨部门测试要求

测试跨部门数据隔离时，需要从数据库获取：
- 不同部门的 dept_manager 用户（至少2个不同部门）
- 不同部门的 engineer 用户（至少2个不同部门）

## 十五、数据清理规则

### 测试前清理

```sql
-- 清理所有E2E测试数据
DELETE FROM tasks WHERE description LIKE 'E2E%';
DELETE FROM projects WHERE name LIKE 'E2E%';
DELETE FROM members WHERE name LIKE 'E2E%';
DELETE FROM departments WHERE name LIKE 'E2E%';
```

### 测试后清理

| 数据类型 | 清理条件 | 清理方式 |
|----------|----------|----------|
| 测试项目 | name LIKE 'E2E%' | 软删除 |
| 测试任务 | description LIKE 'E2E%' | 硬删除 |
| 测试用户 | username LIKE 'E2E%' | 禁用 |
| 测试部门 | name LIKE 'E2E%' | 硬删除 |
